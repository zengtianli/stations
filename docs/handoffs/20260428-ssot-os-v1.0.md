## 踩过的坑（本会话）

### 1. plan §2.3b 字面方案 cross-root member 不工作
`tools/pyproject.toml workspace members=["doctools","../devtools"]` — uv sync 跑了，doctools 走共享 venv ok，但 devtools 自己 `cd && uv run` 仍用 devtools/.venv（uv 上溯找不到 tools workspace root）。教训：plan 字面不一定可达；测了再说。

### 2. UV_PROJECT_ENVIRONMENT 不是"借"是"claim"
设它指 tools/.venv，uv 把 tools/.venv 重建成 devtools 的 venv，**破坏共享**。env 名误导。教训：uv 文档要看实例。

### 3. zsh noclobber 拦截 freeze 备份
`uv pip freeze > /tmp/file.txt` 第二次失败 — zsh `noclobber` 不允许覆盖。用 `>|` 强写。

### 4. 孤儿 gitlink (submodule 残留)
`tools/configs/hammerspoon` 是 mode 160000 gitlink 但 `.gitmodules` 不存在。`git rm` 报"could not lookup name"。需 `git rm --cached` 后物理 rm。

### 5. cc-options venv 真 import 名
plan §2.2 写 `snaptrade_python_sdk`（PyPI 包名）；真 import 是 `snaptrade_client`（module 名）。verified: `/opt/homebrew/bin/python3 -c "import snaptrade_client"` ok。
