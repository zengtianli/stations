# SSH 密钥免密登录

## 原理

- **密码登录**：每次输密码，服务器验证密码
- **密钥登录**：本地生成密钥对（私钥 + 公钥），公钥放服务器，连接时自动验证，无需密码

类比：私钥是钥匙（只在你手里），公钥是锁（装在服务器上），钥匙能开锁就放行。

## 操作步骤

### 1. 生成密钥对（本地 Mac）

```bash
ssh-keygen -t ed25519 -C "备注名" -f ~/.ssh/id_ed25519 -N ""
```

| 参数 | 含义 |
|------|------|
| `-t ed25519` | 加密算法，ed25519 更安全更短 |
| `-C "备注名"` | 公钥末尾的注释，方便识别 |
| `-f 路径` | 指定密钥保存位置 |
| `-N ""` | 私钥密码设为空（直接免密） |

生成两个文件：
- `~/.ssh/id_ed25519` — 私钥（绝对不能泄露）
- `~/.ssh/id_ed25519.pub` — 公钥（可以随意分发）

### 2. 把公钥复制到服务器

```bash
# 方法一：用 ssh-copy-id（推荐）
ssh-copy-id -i ~/.ssh/id_ed25519.pub root@服务器IP

# 方法二：手动复制（ssh-copy-id 失败时用）
cat ~/.ssh/id_ed25519.pub
# 复制输出内容，然后 SSH 登录服务器执行：
mkdir -p ~/.ssh && echo "公钥内容" >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys
```

### 3. 验证

```bash
ssh root@服务器IP
# 不再提示输入密码，直接登录成功
```

## 权限要求

SSH 对文件权限有严格要求，权限不对会拒绝使用：

| 文件/目录 | 权限 | 命令 |
|-----------|------|------|
| `~/.ssh/`（本地和服务器） | 700 | `chmod 700 ~/.ssh` |
| `~/.ssh/id_ed25519`（私钥） | 600 | `chmod 600 ~/.ssh/id_ed25519` |
| `~/.ssh/authorized_keys`（服务器） | 600 | `chmod 600 ~/.ssh/authorized_keys` |

## 常见问题

**Q: 换电脑了怎么办？**
A: 把 `~/.ssh/id_ed25519` 和 `id_ed25519.pub` 复制到新电脑同一位置，或重新生成再推送公钥。

**Q: 一台服务器可以放多个公钥吗？**
A: 可以，`authorized_keys` 文件每行一个公钥，多人/多设备各加一行。

**Q: 密钥登录和密码登录可以共存吗？**
A: 可以。如果想禁用密码登录（更安全），编辑服务器 `/etc/ssh/sshd_config`，设置 `PasswordAuthentication no`，然后 `systemctl restart sshd`。
