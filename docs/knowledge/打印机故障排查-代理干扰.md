# macOS 打印机故障：代理干扰导致无法打印

**日期**：2026-02-27
**设备**：Canon iR C3322L（192.168.32.173）
**系统**：macOS

---

## 症状

- 早上能打印，下午突然不行
- `ping 192.168.32.173` 正常（3ms）
- CUPS 报错：`Unable to get printer status`
- 打印队列卡住（Job 157 持续重试）

## 排查过程

```bash
# 1. ping 通 → 网络层没问题
ping 192.168.32.173  # OK

# 2. 端口全通 → 打印机服务正常
nc -z -w 3 192.168.32.173 9100  # OK (RAW)
nc -z -w 3 192.168.32.173 631   # OK (IPP)
nc -z -w 3 192.168.32.173 80    # OK (HTTP)

# 3. 但 curl 连不上 → HTTP 层有问题
curl -s http://192.168.32.173/  # 返回 000，exit code 7

# 4. 检查代理 → 找到根因
echo $all_proxy  # 127.0.0.1:7890（Clash 代理）
```

## 根因

系统设了 `all_proxy=127.0.0.1:7890`（Clash 代理），但代理服务没运行。

- `ping` 用 ICMP 协议 → **不走代理** → 正常
- CUPS 用 IPP/HTTP 协议 → **走代理** → 代理没开 → 连接失败

## 解决方案

把打印机协议从 `ipp://`（走 HTTP，受代理影响）改成 `socket://`（直连 TCP 9100 端口，不受代理影响）。

```bash
# 1. 清理卡住的任务
cancel -a

# 2. 删除旧打印机
lpadmin -x _192_168_32_173

# 3. 用 socket 协议重新添加（关键！）
lpadmin -p Canon_iR_C3322L \
  -v socket://192.168.32.173:9100 \
  -P /Library/Printers/PPDs/Contents/Resources/CNPZUIRAC3322S.ppd.gz \
  -E

# 4. 设为默认打印机
lpadmin -d Canon_iR_C3322L

# 5. 测试
echo "test" | lp
```

## GUI 操作（等效）

1. **系统设置 → 打印机与扫描仪**
2. 删除现有打印机（点 `-`）
3. 添加打印机（点 `+`）→ 选择 IP 标签页
4. 地址：`192.168.32.173`
5. **协议选 `HP Jetdirect - Socket`**（不要选 IPP）
6. 驱动选 Canon iR C3322L

## 经验教训

1. **ping 通 ≠ 能打印**：ping 用 ICMP 不走代理，打印用 HTTP 走代理
2. **代理是隐形杀手**：`all_proxy` 会影响所有 HTTP 流量，包括打印
3. **socket 协议更可靠**：直连 TCP 端口，不受 HTTP 代理、Cookie、认证等干扰
4. **下次打印机出问题先查**：`echo $all_proxy $http_proxy $https_proxy`
