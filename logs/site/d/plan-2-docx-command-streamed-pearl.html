# 写 eco-flow + reclaim 的 Commands Cheatsheet ## Context panan 标书项目已经完成了"工作 × 命令反查映射" cheatsheet（`~/Work/_playbooks/bids/_commands-cheatsheet.md`），这是用户沉淀方法论的标准产出。现在要给另外两块工作——**生态流量**（`~/Work/eco-flow/`）和**再生水**（`~/Work/zdwp/projects/reclaim/`）——各写一份同样结构的 cheatsheet。 探索已完成，两个领域的工作脉络和已有可复用资产都摸清了。本次只写 MD 文件 + 更新 INDEX，不涉及 skill 抽取或脚本泛化。 ## 探索结论速记 ### eco-flow（生态流量） - **定位**：浙江省小型水库生态流量核定与保障项目 - **5 个已有 archive:* slash commands**：prep-engineering / dissect-report / prep-ecoflow-calc / gen-report + `/prep-basic-info` 和 `/md2word` - **核心脚本**（`code/脚本/core/`）：eco_flow_calc.py / generate_table_2_1.py / quick_filter.py / merge_reservoir_data.py / fill_from_md.py / simplify_fields.py - **数据库**：db_reservoirs.db（4278 座水库）+ db_water_stats.db - **模板**：景宁为标杆，`模板/ref_chapters/` 存标杆章节 - **子项目命名**：`{city}-{county}`（lishui-景宁、taizhou-天台） - **工作 8 阶段**：初始化 → 数据筛选/合并 → 工程排查表 → 生态流量计算 → 基础信息 → 成品报告拆解 → 逐章生成 → Word 输出 ### reclaim（再生水） - **定位**：吴兴区工业集聚区水资源循环梯级利用试点；核心交付 AHP+CRITIC+TOPSIS 水效评估方法 - **可复用 commands**：`/read-docx` `/diff-docx` `/edit-docx` `/merge-md` 等 docx 工具链；`capacity` skill（AHP-CRITIC-TOPSIS 通用方法论） - **脚本**：父项目 `zdwp/scripts/zdwp_health.py` - **版本迭代痛点**：6 个版本散落，MD 拆分未与最新 docx 同步 - **子项目**：`huzhou-吴兴` 按 docs/01-05 + _archive 组织 - **工作 8 阶段**：基线 → 指标体系 → 技术方法迭代 → 企业水效评级 → 动态调整机制 → 成果汇总 → 数据分析 → 路线图 ## 方案 创建 2 份 cheatsheet，格式完全对齐 panan 的： 1. 顶部速查表（工作 × 下次用什么命令） 2. 按阶段展开：每阶段"做过什么 / 下次怎么做" 3. 底部"关联资料"指向 CLAUDE.md / HANDOFF.md / SKILL.md ### 文件 1：`~/Work/_playbooks/eco-flow/_commands-cheatsheet.md` 内容涵盖（以探索发现为准）： - 速查表：8 个阶段对应 6 个 archive/全局 command - 子项目结构模板（`{city}-{county}/{data,deliverables,docs}`） - 标杆样板：lishui-景宁（44 座水库样板） - 踩坑摘要：工程资料依赖人工、频率水资源表需人工摘录、自动化率 60% ### 文件 2：`~/Work/_playbooks/reclaim/_commands-cheatsheet.md` 内容涵盖： - 速查表：8 阶段 → docx 工具链 + capacity skill - 单子项目结构（`huzhou-吴兴/docs/01-05` + `_archive`） - 踩坑摘要：版本管理、MD/docx 同步、references 索引缺失、元数据过期 - 横向复用：列出其他 zdwp 子项目（agri-price / avail-water 等）的共性工作与指标体系复用机会 ### 文件 3：更新 `~/Work/_playbooks/INDEX.md` 在 eco-flow 和 reclaim 两个领域标题下，加"速查手册"链接（与 bids 章节保持一致格式）。 ## 关键文件 | 文件 | 操作 | |------|------| | `~/Work/_playbooks/eco-flow/_commands-cheatsheet.md` | **新建** | | `~/Work/_playbooks/reclaim/_commands-cheatsheet.md` | **新建** | | `~/Work/_playbooks/INDEX.md` | **更新**：eco-flow + reclaim 两节加速查手册链接 | ## 复用 - 格式模板：`~/Work/_playbooks/bids/_commands-cheatsheet.md` 照搬结构 - 探索结论：本计划"探索结论速记"节已含全部必要事实，不需再读业务文件 ## 验证 ```bash cat ~/Work/_playbooks/eco-flow/_commands-cheatsheet.md | head -30 cat ~/Work/_playbooks/reclaim/_commands-cheatsheet.md | head -30 cat ~/Work/_playbooks/INDEX.md ``` ## 不做 - 不抽 skill（这两个领域的 commands 大部分已存在；抽 skill 是 L1 → L2 升级，需 ≥2 次实战，暂无触发） - 不动业务数据 - 不做跨领域脚本迁移（eco-flow 的 eco_flow_calc.py 留在 eco-flow 项目内就好）
