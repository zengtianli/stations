Plan: 关于我页面优化 — 技能精简 + 展望文案修复

PLAN · (plan) · 2026-04-14 15:30 · ~/.claude/plans/distributed-wibbling-aurora.md
# Plan: 关于我页面优化 — 技能精简 + 展望文案修复 ## Context 关于我页面有两个问题： 1. **技能展示（进度条）太傻** — 所有方向都显示全部技能类别（水利8项+ML+软件+工具），indie 方向下出现"生态流量核定""MIKE 系列"等完全无关的技能。进度条百分比形式太"简历模板风"，不 Apple。用户选择"精简保留"：每个方向只保留强相关技能，去掉进度条百分比，改为标签/卡片形式。 2. **未来展望副标题没跟方向走** — indie 方向下还显示"我对未来水利行业发展的思考与个人职业规划"，应该每个方向有对应的 description。 ## 改动清单 ### 1. 技能展示组件重做 **文件**: `components/page-sections.tsx` (SkillsVisual 组件) 去掉进度条百分比，改为标签卡片形式： - 每个技能类别是一张白卡 - 类别名称作为卡片标题 - 技能名用标签（pill/badge）展示，不显示百分比 - 有 projects 字段的技能，下方灰字显示关联项目 ### 2. 技能内容按方向精简 **文件**: `content/about/skills.md` 每个方向只显示强相关的技能类别（通过 tracks overlay 控制）： - **hydro**（默认）：全部显示（水利专业技能、ML、软件开发、专业工具）— 这是本行，保留完整 - **ai**：AI & LLM 工程 + 机器学习与数据科学 + 软件开发（去掉水利专业技能、专业工具） - **devtools**：开发者工具 + 软件开发 + 专业工具中的 Neovim/Docker（去掉水利、ML） - **indie**：产品与运营 + 软件开发 + 量化投资相关（去掉水利专业技能、MIKE 等） 实现方式：在 tracks overlay 里用 `hiddenCategories` 数组指定要隐藏的类别，组件渲染时过滤掉。 ### 3. 未来展望 description 按方向定制 **文件**: `content/about/future.md` 现在 description 只有一个："我对未来水利行业发展的思考与个人职业规划" 各方向 tracks overlay 已有 visionPoints 覆盖，但没覆盖 title 和 description。 添加 description 覆盖： - **hydro**（默认）："我对未来水利行业发展的思考与个人职业规划"（保持） - **ai**：`description: "AI 工程领域的技术方向与实践目标"` - **devtools**：`description: "开发者工具方向的产品理念与技术路线"` - **indie**：`description: "独立开发与量化投资的下一步计划"` 同时覆盖 title： - **hydro**："未来发展与展望"（保持） - **ai**："技术方向" - **devtools**："产品路线" - **indie**："下一步计划" ### 4. indie 方向 skills 增加"量化投资"类别 **文件**: `content/about/skills.md` (tracks.indie.extraCategories) 在现有"产品与运营"之后增加： ```yaml - name: "量化投资" skills: - name: "期权策略 (Covered Call)" level: 90 - name: "跨境资产配置" level: 90 - name: "Greeks 风控" level: 85 - name: "Streamlit 仪表盘" level: 90 ``` ## 关键文件 | 文件 | 改动 | |------|------| | `components/page-sections.tsx` | SkillsVisual 组件重做（标签替代进度条） | | `content/about/skills.md` | tracks overlay 加 hiddenCategories + indie 加量化投资类别 | | `content/about/future.md` | tracks overlay 加 title + description 覆盖 | | `lib/track-content.ts` | resolveTrackContent 已支持字段覆盖，无需改动 | | `app/about/page.tsx` | 可能需要传 hiddenCategories 过滤逻辑 | ## 验证 1. `pnpm build` 通过 2. 检查 4 个方向的关于页： - hydro：技能全显示，展望标题"未来发展与展望" - ai：只显示 AI/ML/软件，展望标题"技术方向" - devtools：只显示工具/软件，展望标题"产品路线" - indie：显示产品运营/量化投资/软件，展望标题"下一步计划" 3. 技能以标签形式展示，无进度条百分比
