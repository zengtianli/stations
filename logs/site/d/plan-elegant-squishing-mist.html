# Plan: 拆分 scripts 仓库为 4 个独立 repo ## Context `~/Dev/scripts/` 是一个 40+ 脚本的 monorepo，涵盖文档处理、网络代理、开发工具、macOS 效率工具等完全不同的领域。用户决定将其完全拆分为 4 个独立 repo，老 scripts repo 退役。 ## 拆分方案 | 新 repo | 来源 | 脚本数 | Raycast wrappers | |---------|------|--------|-----------------| | **doctools** | document/ + data/ | 17 | 27 | | **devtools** | tools/ + repo_manager + lib/tools/ | ~10 | 7 (含 cclog) | | **mactools** | file/ + system/ + window/ | 11 | 13 | | **clashx** | network/ | 6 | 6 | ## 各 repo 目录结构 所有 repo 保留 `scripts/category/` + `lib/` + `raycast/` 结构，使 `common.sh` 和 `run_python.sh` 的相对路径逻辑无需修改。 ### 1. ~/Dev/doctools/ ``` scripts/{document/, data/} # 原封不动 lib/ # display, file_ops, finder, progress, docx_xml, clipboard, env, usage_log, __init__, common.sh, llm_client raycast/{commands/, lib/} # 27 wrappers templates/ # template.docx (gitignored) README.md, CLAUDE.md, ruff.toml, requirements.txt, .gitignore ``` ### 2. ~/Dev/devtools/ ``` scripts/tools/ # 原封不动（含 llm_client、hooks、printer/） lib/ # display, file_ops, env, usage_log, __init__, common.sh lib/tools/ # cf_api, gen_claude_md, health_check, sync_index, validate_raycast, vps_cmd repo_manager.py # 根目录 projects/company_query/ # 独立项目 raycast/{commands/, lib/} # 7 wrappers (app_open, git_smart-push, sec_export, tts + 3 cclog + 5 hy_*) README.md, CLAUDE.md, ruff.toml, requirements.txt, .gitignore ``` ### 3. ~/Dev/mactools/ ``` scripts/{file/, system/, window/} # 原封不动 lib/ # display, file_ops, finder, clipboard, env, usage_log, __init__, common.sh, llm_client raycast/{commands/, lib/} # 13 wrappers README.md, CLAUDE.md, .gitignore ``` ### 4. ~/Dev/clashx/ ``` scripts/network/ # 原封不动 lib/ # clashx.sh, common.sh raycast/{commands/, lib/} # 6 wrappers README.md, CLAUDE.md, .gitignore ``` ## 关键依赖处理 ### lib/ 分配 | 模块 | doctools | devtools | mactools | clashx | |------|----------|----------|----------|--------| | display.py | Y | Y | Y | - | | file_ops.py | Y | Y | Y | - | | finder.py | Y | - | Y | - | | progress.py | Y | - | - | - | | docx_xml.py | Y | - | - | - | | clipboard.py | Y | - | Y | - | | env.py | Y | Y | Y | - | | usage_log.py | Y | Y | Y | - | | common.sh | Y | Y | Y | Y | | clashx.sh | - | - | - | Y | 每个 repo 的 `lib/__init__.py` 只 re-export 该 repo 存在的模块。 ### llm_client.py (80 行，wraps `claude -p`) - **devtools**: 原位保留 `scripts/tools/llm_client.py` - **doctools**: 复制到 `lib/llm_client.py`，3 个脚本 import 改为 `from llm_client import chat` - `document/bullet_to_paragraph.py` - `document/md_tools.py` (frontmatter 子命令) - `document/scan_sensitive_words.py` - **mactools**: 复制到 `lib/llm_client.py`，1 个脚本 import 改为 `from llm_client import chat` - `file/smart_rename.py` ### common.sh / run_python.sh 路径逻辑全部基于相对路径 (`$_COMMON_SH_DIR/..`)，目录结构保留一致 → **无需改动**。 ### 清理 -
