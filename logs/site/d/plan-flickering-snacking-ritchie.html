# Multi-Track 个人站实现计划 ## Context 当前 tianlizeng.cloud 是纯水利方向的个人站。用户希望同一个网站能服务 4 个就业方向：AI 工程、开发工具、水利数字化、独立开发。访客选择方向后，Hero 文案、核心能力、技能排序、项目筛选、博客过滤全部跟着变。水利方向是默认值，保持现有 SEO 和行为不变。 ## 4 个 Track | ID | 标签 | 英文 | |----|------|------| | `hydro` | 水利数字化 | Water Resources（默认） | | `ai` | AI 工程 | AI Engineering | | `devtools` | 开发工具 | DevTools | | `indie` | 独立开发 | Indie Dev | --- ## Phase 1: Track 基础设施 ### 1.1 新建 `lib/track.ts` — Track 类型定义 + 元数据 ```ts export const TRACKS = ['hydro', 'ai', 'devtools', 'indie'] as const export type Track = typeof TRACKS[number] export const DEFAULT_TRACK: Track = 'hydro' export const TRACK_COOKIE = 'career-track' export const TRACK_META: Record<Track, { label, icon, color }> ``` ### 1.2 新建 `lib/track-server.ts` — 服务端读取 track - 读取顺序：URL `?track=` → cookie → DEFAULT_TRACK - 使用 Next.js 15 的 `cookies()` (async) ### 1.3 新建 `components/track-provider.tsx` — 客户端 Context - 仿照现有 `ThemeProvider` 模式 - `setTrack()` 同时更新 cookie + URL query + `router.refresh()` - 给子组件提供 `useTrack()` hook ### 1.4 修改 `app/layout.tsx` - 在 `ThemeProvider` 内嵌套 `TrackProvider` - 服务端读 cookie 传入 `initialTrack` ### 1.5 新建 `components/track-switcher.tsx` — 方向选择器 UI - 紧凑 pill 组，4 个方向，当前高亮 - 桌面端：放 navbar 搜索图标左侧 - 移动端：navbar 移动菜单顶部 ### 1.6 修改 `components/navbar.tsx` - 引入 `<TrackSwitcher />` **验证点**：切换 track 后 URL 带 `?track=ai`，刷新保持，cookie 持久。默认 hydro 无 query param。 --- ## Phase 2: 内容层适配 ### 2.1 新建 `lib/track-content.ts` — Track 内容解析器 ```ts // 将带 tracks overlay 的 metadata 解析为当前 track 的最终值 resolveTrackContent<T>(metadata: T & { tracks?: ... }, track: Track): T // 数组重排序（技能分类等） resolveTrackArray<T>(items: T[], order: string[]): T[] ``` ### 2.2 修改 `lib/profile-config.ts` — 添加 track overlay 给 `heroConfig` 和 `strengthsConfig` 加 `tracks` 字段： **heroConfig.tracks:** - `ai`: tagline="AI 工程 | LLM 工具链架构师", description 强调 CC harness/MCP/Agent 工作流 - `devtools`: tagline="开发者工具产品工程师", description 强调 CLI/Raycast/自动化 - `indie`: tagline="全栈独立开发者", description 强调一人闭环能力 **strengthsConfig.tracks:** - 每个 track 3 张核心能力卡（不同 icon/title/description） ### 2.3 修改 `content/home/hero.md` — 添加 tracks overlay 同 profile-config，subtitle/description 按 track 变化。 ### 2.4 修改 `content/about/intro.md` — 添加 tracks overlay subtitle/description/slogan 按 track 变化。 ### 2.5 修改 `content/about/skills.md` — 添加 track 排序 ```yaml tracks: ai: categoryOrder: ["机器学习与数据科学", "软件开发", "水利工程专业技能", "专业工具"] # 新增 AI 专属分类 extraCategories: - name: "AI & LLM 工程" skills: [{ name: "Claude Code Harness", level: 95 }, ...] devtools: categoryOrder: ["软件开发", "专业工具", ...] ``` ### 2.6 修改 `content/about/future.md` — 添加 track visionPoints 每个 track 3 个不同的愿景点。 ### 2.7 修改 `content/home/latest-updates.md` — 添加 track items 每个 track 展示不同的最新动态。 ### 2.8 修改项目 frontmatter — 添加 track 映射 每个项目 md 文件加 `tracks` 字段： ```yaml tracks: hydro: { highlight: true, featured: true } devtools: { brief: "Streamlit → Tauri 迁移实践" } indie: { highlight: true } ``` 无 `tracks` 字段的项目在所有方向都展示。 ### 2.9 修改 `lib/content.ts` — 添加 track-aware 查询函数 ```ts getAllProjectsForTrack(track): Promise<ProjectContent[]> getBlogPostsForTrack(track): Promise<BlogPost[]> getLatestUpdatesForTrack(track): Promise<LatestUpdatesContent> ``` 博客按 tag 过滤：定义 `TRACK_BLOG_TAGS` 映射表。 **验证点**：`getContent()` 原有接口不变，新函数在其基础上叠加 track 解析。 --- ## Phase 3: 组件适配 ### 3.1 修改 `components/hero-section.tsx` - 从直接 import `heroConfig` → 接收 `config` props - 签名: `HeroSection({ config }: { config: HeroConfig })` ### 3.2 修改 `components/page-sections.tsx` — StrengthsSection - 从直接 import `strengthsConfig` → 接收 `strengths` props - 签名: `StrengthsSection({ strengths }: { strengths: StrengthConfig[] })` - 其他组件（AboutIntro, SkillsVisual, FutureOutlook, Timeline）已接受 props，无需改签名 ### 3.3 修改 `app/page.tsx` — 首页 ```ts export default async function Home({ searchParams }) { const track = await getServerTrack(await searchParams) const heroResolved = resolveTrackContent(heroConfig, track) const strengthsResolved = getTrackStrengths(track) const updates = await getLatestUpdatesForTrack(track) return ( <HeroSection config={heroResolved} /> <StrengthsSection strengths={strengthsResolved} /> <LatestUpdates data={updates} /> ) } ``` ### 3.4 修改 `app/about/page.tsx` - 读 track，resolve intro/skills/future 内容后传给组件 - skills 支持 categoryOrder 重排 + extraCategories 合并 ### 3.5 修改 `app/projects/page.tsx` - 调用 `getAllProjectsForTrack(track)` 替代 `getAllProjects()` ### 3.6 修改 `app/blog/page.tsx` - 调用 `getBlogPostsForTrack(track)` 替代 `getAllBlogPosts()` ### 3.7 详情页不变 `app/projects/[slug]` 和 `app/blog/[slug]` 保持 SSG，不依赖 track。 ### 3.8 修改 `lib/seo-config.ts` - 添加 `getTrackMeta(track)` 返回 track 特定的 title/description **验证点**：默认 hydro track 下网站行为和外观完全不变。切换 track 后 Hero/Strengths/Skills/Projects/Blog 均跟随变化。 --- ## Phase 4: 新内容创建 ### 4.1 新项目页面（content/projects/items/） | 文件 | 项目 | 主要 Track | |------|------|-----------| | `cc-harness.md` | Claude Code Harness 体系（43 commands, 14 skills, hooks, memory） | ai, devtools | | `devtools-suite.md` | 开发者工具四件套（doctools/devtools/mactools/clashx） | devtools, indie | | `briefing-system.md` | 每日 Briefing 自动推送系统 | ai, devtools | | `personal-website.md` | 本站（Next.js + Multi-track） | devtools, indie | | `vps-infra.md` | VPS 基础设施（24 systemd + 4 Docker + CF） | devtools, indie | ### 4.2 新博客文章（content/blog/） 1. `claude-code-harness-architecture.md` — CC harness 深度解析（tags: AI, LLM, 开发工具） 2. `streamlit-to-tauri.md` — Streamlit → Tauri 迁移经验（tags: 开发工具, 独立开发） 3. `one-person-infra.md` — 一个人的全栈基础设施（tags: 独立开发, DevOps） ### 4.3 现有项目 track 标注 | 现有项目 | hydro | ai | devtools | indie | |---------|-------|-----|---------|-------| | hydro-capacity | primary | - | secondary | primary | | hydro-efficiency | primary | secondary | - | primary | **验证点**：每个 track 至少有 3 个 featured 项目。 --- ## Phase 5: Resume 集成（可选，后续做） - `lib/resume-builder.ts` 添加 `ai`/`devtools`/`indie` 模板 - Resume 页面按当前 track 预选模板 --- ## 实现顺序 1. Phase 1（基础设施）→ 能切换但不影响内容 2. Phase 3.1-3.2（组件签名改造）→ HeroSection/StrengthsSection 接受 props 3. Phase 2.1-2.2（track-content + profile-config overlay）→ 首页能响应 track 4. Phase 3.3（首页接入）→ 首页完整可用 5. Phase 2.3-2.9（所有内容层 overlay）+ Phase 3.4-3.8（所有页面接入） 6. Phase 4（新内容） ## 关键文件清单 | 操作 | 文件 | |------|------| | 新建 | `lib/track.ts`, `lib/track-server.ts`, `lib/track-content.ts` | | 新建 | `components/track-provider.tsx`, `components/track-switcher.tsx` | | 修改 | `app/layout.tsx`, `app/page.tsx`, `app/about/page.tsx`, `app/projects/page.tsx`, `app/blog/page.tsx` | | 修改 | `components/hero-section.tsx`, `components/page-sections.tsx`, `components/navbar.tsx` | | 修改 | `lib/profile-config.ts`, `lib/content.ts`, `lib/seo-config.ts` | | 修改 | `content/home/hero.md`, `content/about/intro.md`, `content/about/skills.md`, `content/about/future.md`, `content/home/latest-updates.md` | | 修改 | `content/projects/items/*.md`（所有项目加 tracks） | | 新建 | 5 个新项目 md + 3 篇新博客 md | ## 验证方案 1. `pnpm build` 无报错 2. `pnpm dev` 下： - 默认打开 → hydro 方向，和改动前完全一致 - `?track=ai` → Hero/Strengths/Skills/Projects/Blog 全部切换为 AI 方向内容 - 点击 TrackSwitcher 切换 → URL 更新，刷新保持 - 项目详情页/博客详情页 → 不受 track 影响 3. Cookie 持久：关闭页面重新打开，track 保持
