# Plan · SSOT 盲区补救（v0.1 → v1.0 一次性合并） ## Context `/refactor-dir` 走文本类引用是 0 死链了，但今天测试暴露 3 类盲区： 1. **Symlink target 不跟随 migrations** — 已发现 5 个真实命中：`~/Dev/Work/eco-flow/.claude/commands/{dissect-report,prep-basic-info}.md` 指向 `~/Dev/cc-configs/...`、`Work/zdys/irrigation` 指向 `~/Dev/hydro-irrigation`、`Work/risk-map/{qgis,code}` 指向 `~/Dev/hydro-{qgis,risk}`。这些 dangling 但 migrations 全有，rebuild-symlinks 一次能修。 2. **`paths.yaml` 自身的 `allow_missing:` 不跟 migrations 走** — 改名后老前缀挡不住，原本被挡的"隐性死链"显形。 3. **运行时拼装路径** — `f"~/Dev/{name}"` 静态扫不到（station-promote.sh / menus.py 的合法模板就是这样写的）。只能 audit-only 警告 + 文档说明。 HANDOFF 路线图 v0.1→v0.2→v0.3→v1.0 用户确认**一次性合并完成**（不再分轮 handoff）。 ## 当前状态（实测） - `paths.py audit --brief` → `55 registered / 0 dead / 0 drift` ✓ - `~/Dev` 下源码 symlink 70 个（已排除 .venv），其中 dangling 22 个：5 个 migration-hit（v0.1 修目标）、~17 个外部/历史孤儿 - `~/Dev/stations/docs` 已恢复（不在 docs-renamed-test 状态） ## Scope 四个增量 + 一份 final HANDOFF： | # | 文件 | 增量 | |---|---|---| | A | `~/Dev/devtools/lib/tools/paths.py` | 加 `scan-symlinks` / `rebuild-symlinks` / `rewrite-allow-missing` / `audit-runtime` 4 个子命令；`audit` 第 5 项加 symlink check（仅当 brief 输出层面） | | B | `~/Dev/tools/cc-configs/commands/refactor-dir.md` | 在 Step 6 后插 6.5 / 6.6 / 6.7 三步（rewrite-allow-missing → rebuild-symlinks --apply → scan-symlinks --strict） | | C | `~/Dev/tools/configs/playbooks/paths.md` | 加 §「symlink 维护」+ §「运行时拼装路径限制」 | | D | `~/Dev/HANDOFF-ssot-blindspots-v1.0-2026-04-25.md` | 4 轮全合并的 final handoff，关闭 v0.1 那份 | ## 实现细节 ### A. paths.py 4 个新子命令 #### `scan-symlinks [--strict] [--json]` - **扫描范围**：`rglob("*")` ~/Dev，仅 `is_symlink()` 项；目录排除复用 `SCAN_EXCLUDE_DIRS`，但**额外移除 `Work` / `_archive`**（这些目录有真实 symlink 需要修），再加 `.venv / venv / __pycache__ / .turbo / .vercel / .next / .cache / .pytest_cache / .mypy_cache / .ruff_cache` - **每条 symlink 分类**： - `OK` — `link.exists()` 真 - `DANGLING — migration hit` — target 命中某 migration `from` 前缀，能算出新 target - `DANGLING — allow_missing` — target 命中 `allow_missing` 前缀 - `DANGLING — orphan` — 其他 dangling - `RELATIVE` — target 是相对路径（v0.1 不处理，提示用户手动） - **`--strict` 退出 1**：仅当存在 `migration hit` 但未修复的；orphan/allow_missing 只报告不阻塞（避免被历史债噎死） - **算法**：target 既可能是 `~/...`、绝对 `/Users/tianli/...`、`$HOME/...` 三种，逐一应用 `_build_rewrite_patterns` 现成正则；最长前缀优先匹配 #### `rebuild-symlinks [--dry-run] [--apply] [--backup]` - 调 `scan-symlinks` 拿 `migration hit` 列表 - 对每条：删旧 symlink、用 `os.symlink(new_target, link_path)` 重建（保留 target 原写法形态：`~/...` 写法保留 `~/...`，绝对保留绝对，避免无关变形） - `--backup` 把 `<link_path>\t<old_target>\t<new_target>` 追加进 `~/Dev/_archive/symlink-rewrites/<ts>.tsv`（恢复用） - 默认 `--dry-run`，必须 `--apply` 才动手 - 之后自动 `scan-symlinks --strict` 自检 #### `rewrite-allow-missing [--dry-run] [--apply]` - 行级解析 `paths.yaml`（不 dump-back，避免丢注释/顺序） - 定位 `allow_missing:` 段落（next top-level key 之前） - 对每行 `- path: <value>` 应用 migrations 前缀替换；其他行不动 - `--apply` 写回 `paths.yaml`；`--dry-run` 打 diff - 注意 `paths.yaml` 在 `SCAN_EXCLUDE_FILENAMES`（rewrite-dead 不动它），所以这是独立路径 #### `audit-runtime` - 启发式 grep：`re.compile(r'(?:f["\']|".*\+\s*)(?:~/Dev|/Users/tianli|\$HOME)[^"\']*\{')` 抓 `f"~/Dev/{x}"` / `"$HOME/" + x` 形态 - 扫 `*.py / *.sh / *.ts / *.tsx`，复用 `_iter_scan_files` - **只输出列表 + 总数，不阻塞，不 exit 1**（合法模板就是这样写的） - 顶部一行说明："如下命中是运行时拼装路径，paths_const 静态扫不到。建议消费者从 paths_const 取根再拼，但 station-promote / menus 之类合法模板可忽略。" #### `audit` 增量 - 追加第 5 项：`symlinks` — 扫 migration-hit DANGLING 数量；写进 `drift_candidates` - `--brief` 输出沿用单行格式（不破坏现有消费者），但 `drift` 计数会涵盖 symlink ### B. refactor-dir.md 编排升级 在现有 Step 6 (`build-const`) 后、Step 7 (`scan-dead`) 前插： ``` Step 6.5 · rewrite-allow-missing --apply → 把 allow_missing: 老前缀按 migrations 平滑迁移 Step 6.6 · rebuild-symlinks --apply → 把 migration-hit dangling symlinks 重指新 target Step 6.7 · scan-symlinks --strict → 验证 0 migration-hit dangling（orphan 不阻塞） ``` 失败回滚增量： - 6.5 失败：从 backup 撤回 paths.yaml 的 allow_missing 修改 + 同 Step 5 失败处理 - 6.6 失败：用 `~/Dev/_archive/symlink-rewrites/<ts>.tsv` 反向重建 + 同 Step 5 - 6.7 失败：打印详情让用户判断（同 Step 7 失败语义） ### C. paths.md playbook 补两节 - §「symlink 维护」：列出 scan-symlinks / rebuild-symlinks 命令、何时跑（refactor-dir 已自动）、故障定位 - §「运行时拼装路径限制」：解释 audit-runtime 是 informational、消费者写法建议（先取根再拼子） ### D. HANDOFF v1.0 写 `~/Dev/HANDOFF-ssot-blindspots-v1.0-2026-04-25.md`，结构： - Recap：3 类盲区 + 一次合并落地 - 落地清单：A/B/C 增量 + 修了 5 个真实 dangling - 验收结果（实测数字） - v1.0 之后：明确 roadmap 关闭，新主题另起 v2.0 最后**删除 v0.1 那份**（`HANDOFF-ssot-blindspots-v0.1-2026-04-25.md`），归档需要的话搬 `_archive/` —— 用户再决定。 ## Verification（按顺序跑） 1. **修代码后单元功能**： -
