# Plan: Skills 重命名 + resources skill 新建 ## Context harness 系统已就绪（~/Dev/cc-harness/ symlink 到 ~/.claude/）。现在要： 1. 去掉 `zdwp-` 前缀——项目已独立，前缀多余 2. 新建 `resources` skill——让 CC 知道共享数据在哪 ## Step 1: 重命名 skills（去 zdwp- 前缀） 在 `~/Dev/cc-harness/skills/` 中 rename： | 现名 | 新名 | |------|------| | zdwp-context → context | | zdwp-structure → structure | | zdwp-plan-first → plan-first | | zdwp-quarto → quarto | | zdwp-eco-flow → eco-flow | | zdwp-risk-map → risk-map | | zdwp-zdys → zdys | | zdwp-water-source → water-src | | zdwp-water-quality → water-quality | 每个 SKILL.md 的 frontmatter `name:` 字段也同步更新。 通过 symlink 自动生效到 ~/.claude/skills/。 ## Step 2: 更新 harness.yaml ```yaml source: ~/Dev/cc-harness/skills global_skills: - context - structure - plan-first - quarto projects: risk-map: path: ~/Work/risk-map skills: [risk-map, resources, plan-first] eco-flow: path: ~/Work/eco-flow skills: [eco-flow, resources, plan-first] zdys: path: ~/Work/zdys skills: [zdys, resources, plan-first] water-src: path: ~/Work/water-src skills: [water-src, resources, plan-first] standalone: - water-quality ``` 注意：`resources` 和 `plan-first` 加到所有水利项目，让每个项目开箱就有数据路径和操作规范。 ## Step 3: 新建 resources skill 创建 `~/Dev/cc-harness/skills/resources/SKILL.md`： ```markdown --- name: resources description: 公司级共享资源目录。当需要查找水库数据、GIS底图、行政边界、水资源年报、行业规范时触发。 --- # 共享资源 (~/Work/resources/) ## 数据 (data/) | 路径 | 内容 | |------|------| | `data/全省水资源基础数据/原始数据/Excel/` | 全省水库、河流、水闸基础数据 xlsx | | `data/全省水资源基础数据/原始数据/GeoJSON/` | 水利要素 GeoJSON | | `data/全省水资源基础数据/原始数据/Shapefile/` | 水利要素 shp | | `data/全省水资源基础数据/工程数据/` | 工程筛选和统计 | | `data/原始年报/` | 2019-2024 全省水资源分区基本情况表 xlsx | ## GIS (gis/) | 路径 | 内容 | |------|------| | `gis/水利要素/水库/` | 水库点位 shp | | `gis/水利要素/` | 河流、水闸等水利要素 | | `gis/行政边界乡镇/` | 乡镇级行政区划 shp | | `gis/专题数据/洪水风险/` | 洪水风险专题 | | `gis/专题数据/岸线规划/` | 岸线规划专题 | | `gis/专题数据/生态流量/` | 生态流量专题 | | `gis/专题数据/供水管网/` | 供水管网专题 | | `gis/小水电/` | 小水电 geojson | | `gis/其他.gdb/` | ArcGIS GDB（杂项空间数据）| ## 参考资料 (references/) | 路径 | 内容 | |------|------| | `references/规范政策/` | 行业规范、政策文件 | | `references/规划报告/` | 规划报告 | | `references/内部资料/` | 公司内部模板、字体、软著 | | `references/数据资源/` | 数据资源索引 | ## 使用方式 读取共享数据时用绝对路径 `~/Work/resources/...`。 GIS 数据坐标系统一为 CGCS2000。 ``` ## Step 4: 更新已分发的项目 skills 运行 `harness sync --force` 将重命名后的 skills 推送到各项目。 旧的 zdwp-* 目录在各项目的 .claude/skills/ 中需要清理（sync 只加不删，需要手动删旧名）。 ## Step 5: 更新 zdwp 中的引用 - `~/Work/zdwp/.claude/skills/` 中的原始 zdwp-* skills 可以保留（zdwp 项目本身还用），也可以同步改名 - `~/Work/zdwp/CLAUDE.md` 中 Skills 段引用更新 ## 关键文件 | 文件 | 操作 | |------|------| | `~/Dev/cc-harness/skills/zdwp-*/` | rename 目录 + 更新 SKILL.md name 字段 | | `~/Dev/cc-harness/skills/resources/SKILL.md` | 新建 | | `~/Dev/cc-harness/harness.yaml` | 更新 skill 名 + 加 resources | | `~/Dev/cc-harness/harness.py` | 不需改（目录名变了但逻辑不变） | | `~/Work/{risk-map,eco-flow,zdys,water-src}/.claude/skills/` | sync 后更新 + 删旧名 | ## 验证 ```bash # 1. 重命名后全局 skills 加载正常 ls ~/.claude/skills/ # 应该看到 context, structure, plan-first, quarto, ... # 2. status 正常 python3 ~/Dev/cc-harness/harness.py status # 3. sync 到各项目 python3 ~/Dev/cc-harness/harness.py sync --force # 4. init 测试——看到 resources 模块挂载 python3 ~/Dev/cc-harness/harness.py init ~/Work/eco-flow # 期望输出: # ✓ skill: eco-flow # ✓ skill: resources # ✓ skill: plan-first ```
