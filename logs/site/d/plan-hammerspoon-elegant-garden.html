HS 收尾：修 bug + 简化 HUD + 清除 ClashX 模块

PLAN · (plan) · 2026-04-18 19:02 · ~/.claude/plans/hammerspoon-elegant-garden.md
# HS 收尾：修 bug + 简化 HUD + 清除 ClashX 模块 ## Context HS reload 日志显示 18 个快捷键注册成功，但 `init.lua:41` 调用的 `export_config.export()` 崩溃——`~/Dev/ops-console/data/hs_config.json` 没被更新，ops-console dashboard 还在显示旧数据。 同时用户： 1. HUD 三段（图标 + 模块 + 消息）多一段 → 简化为两段（图标 + 消息） 2. 完全不用 ClashX/network 模块 → 整块删掉 ## 根因诊断 ### Bug 1: `export_config.lua:72` 崩溃 ```lua table.insert(modules, file:gsub("%.lua$", "")) ``` Lua 5.3+ 里 `gsub` 返回 **两个值** `(new_str, count)`，作为变参展开给 `table.insert` → 变成 3 参数 `table.insert(t, str, count)`，被当成 `table.insert(t, pos, value)` 解释，pos 必须是 number 但给了 string → 崩溃。 ### Bug 2: HUD 三段冗余 `lib/utils.lua:20`：`"%s %s · %s"` = icon + module + message。 用户选择：**去掉 module，保留 icon + message**。示例： - `✅ 播放/暂停` - `✅ 已打开 Dev` - `❌ ClashX Pro 未运行` ## 修改清单 ### 文件 0 · 删除 ClashX/network 相关代码 1. **删除** `~/.hammerspoon/modules/network.lua`（整个文件，149 行） 2. **`~/.hammerspoon/init.lua` line 35-37** 删除： ```lua -- 初始化代理状态 Menubar 指示器 local network = require("modules.network") network.init_menubar() ``` 3. **`~/.hammerspoon/lib/export_config.lua` line 89** 删除那条 passive feature： ```lua ' {"name": "代理状态指示器", "module": "network", "desc": "Menubar 显示 ClashX 代理模式"}' ``` 并把前一行 line 88 末尾的 `,` 去掉（变最后一项）。 4. `hotkeys.lua` 已经不引用 network（上轮已移除 6 条 ClashX 绑定），无需动。 5. `modules/` 目录扫描（export_config.lua line 69-75）自动不再把 network 放进 modules 列表。 ### 文件 1 · `~/.hammerspoon/lib/export_config.lua:72` ```lua -- before table.insert(modules, file:gsub("%.lua$", "")) -- after (外面包一层括号截断到单返回值) table.insert(modules, (file:gsub("%.lua$", ""))) ``` ### 文件 2 · `~/.hammerspoon/lib/utils.lua:18-21` 改 `display` 函数的格式化： ```lua function M.display(module, message, kind) local cfg = DISPLAY_CONFIG[kind or "info"] or DISPLAY_CONFIG.info hs.alert.show(string.format("%s %s", cfg.icon, message), cfg.duration) -- module 参数保留（签名不变），但仅用于 log 调试 print(string.format("[%s] %s", module, message)) end ``` 保留 `module` 参数的理由：~50 个 call site 全都传了，改签名是churn；log 到 HS console 方便将来调试。 不动 call sites（media.lua / network.lua / window.lua / system.lua / apps.lua / init.lua）。 ## 验证 1. 保存文件后 HS `pathwatcher` 自动 `hs.reload()` 2. 打开 HS Console（菜单栏 → Console）检查： - 无 error trace（特别是原 export 那条 stack traceback 消失） - `[Export] 配置已导出到 /Users/tianli/Dev/ops-console/data/hs_config.json` - 菜单栏 ClashX 状态图标消失 3. 按 `⌘⌃⇧;` 播放/暂停 → HUD 显示 `✅ 播放/暂停`（两段） 4. 按 `⌘⌥,` 打开系统设置 → HUD 显示 `✅ 打开系统设置` 5. 刷新 `https://dashboard.tianlizeng.cloud/hammerspoon`： - 18 条快捷键 - 模块列表不含 `network` - passive_features 只剩 3 条（无"代理状态指示器"） 6. `ls -la ~/Dev/ops-console/data/hs_config.json` mtime 是当前时间 ## 不在本轮范围 - 删除其他 modules/*.lua 中已不绑快捷键的**死函数**（`apps.open_ide`、`apps.launch_apps`、`system.compress/display_1080/display_4k/brew_maintain/file_print/restartApp`、`system.copy_names_and_content`、`window.toggle_float/toggle_mouse/organize/toggle_yabai`）—— Raycast 迁移后再一并清理 - keymap 三键加 HUD（已确认不加） - `show_help` 格式调整（直接调 `hs.alert.show`，不走 display）
