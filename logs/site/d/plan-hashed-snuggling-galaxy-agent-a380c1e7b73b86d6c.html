# Website Redesign Implementation Plan ## Overview Apple-inspired minimal redesign of Next.js 15 portfolio site. Dark-only, Inter-only, no glow/neon effects, new homepage with 4 direction cards, 4 new direction sub-pages, cookie-based track persistence. --- ## Phase 1: CSS Foundation & Theme Cleanup **Goal**: Establish the new visual foundation. Everything after this phase renders in the new style. **Deployable**: Yes — existing pages get new colors/fonts immediately. ### 1.1 `app/globals.css` — Complete rewrite - Change `:root` background to `#0A0A0A` (HSL ~0 0% 4%) - Remove all `:root` light theme variables (there is no `.light` block currently, but remove the `--track-hero-from/to` colored gradients) - Update `--accent` to neutral white for homepage (colorless default) - Update `[data-track="hydro"]` accent to `#60A5FA` (HSL ~213 97% 87%), ai to `#A78BFA`, devtools to `#34D399`, indie to `#FBBF24` - Remove `--track-glow`, `--track-hero-from`, `--track-hero-to` variables entirely - Remove `.hero-gradient` class - Remove `.card-hover` glow shadow (`rgba(100,255,218,0.15)`) - Remove `.timeline-dot` glow shadow - Remove `.typing-effect`, `@keyframes typing`, `@keyframes blink-caret` - Remove `.particles-container` - Keep `@keyframes fadeInUp` and `@keyframes fadeIn` (used for subtle animations) - Remove `@keyframes slideDown` (unused after navbar change) - Change `h1,h2..h6` from `@apply font-montserrat` → remove (Inter inherits) - Change `h1` from `@apply font-orbitron` → remove - Update `.markdown-content h1..h6` to remove `font-montserrat` - Update `--radius` to `1rem` (16px = rounded-2xl) - Update `--card` to a subtle white/5 equivalent in HSL - Remove `--border` visible borders, set to transparent or very subtle ### 1.2 `tailwind.config.ts` — Simplify - Remove `fontFamily.montserrat`, `fontFamily.exo2`, `fontFamily.orbitron` - Keep only `fontFamily.sans: ['var(--font-inter)']` - Remove animation delay plugin (or keep if still useful for fadeInUp stagger) - Keep `fade-in` and `slide-down` keyframes (rename slide-down to fade-up if desired) - Update `--radius` default ### 1.3 `app/layout.tsx` — Font & theme cleanup - Remove `Montserrat`, `Exo_2`, `Orbitron` imports from `next/font/google` - Keep only `Inter` - Remove `${montserrat.variable} ${exo2.variable} ${orbitron.variable}` from body className - Change `<ThemeProvider attribute="class" defaultTheme="dark" enableSystem disableTransitionOnChange>` to `<ThemeProvider attribute="class" defaultTheme="dark" forcedTheme="dark" disableTransitionOnChange>` - OR simply remove ThemeProvider entirely and hardcode `class="dark"` on `<html>` — simpler approach - Keep `<TrackProvider>` wrapping ### 1.4 `lib/design-system/tokens.ts` — Update tokens - Remove `shadows.glow` (`rgba(100,255,218,0.3)`) - Update font family references **Files touched**: `app/globals.css`, `tailwind.config.ts`, `app/layout.tsx`, `lib/design-system/tokens.ts` --- ## Phase 2: Navbar & Footer Refresh **Goal**: Remove TrackSwitcher from navbar, add direction color line, refresh footer. **Deployable**: Yes — nav looks clean, TrackSwitcher is gone. ### 2.1 `components/navbar.tsx` — Redesign - Remove `import { TrackSwitcher }` and all TrackSwitcher usage (desktop bar + mobile menu) - Remove `font-orbitron` from brand link — just `text-xl font-semibold text-white` - Remove the entire "Track Switcher Bar" `<div>` section (lines 87-93) - Remove TrackSwitcher from mobile menu - Add thin colored line at bottom of navbar when user is in a direction: - Read track from `useTrack()` context - When `pathname` starts with `/hydro`, `/ai`, `/devtools`, `/indie` OR when track is set (not on homepage), show a 2px bottom border in the track's color - Use hardcoded color map: `{ hydro: '#60A5FA', ai: '#A78BFA', devtools: '#34D399', indie: '#FBBF24' }` - On homepage (`pathname === '/'`), no color line - Simplify scrolled state styling to match minimal aesthetic ### 2.2 `components/footer.tsx` — Visual refresh - Change `bg-secondary/50` → `bg-white/[0.02]` or just transparent with top border `border-t border-white/5` - Remove `text-accent` from buttons, use `text-white/60 hover:text-white` - Keep same structure, just tone down colors ### 2.3 `components/track-switcher.tsx` — Keep file but mark deprecated - File stays for reference but is no longer imported anywhere - OR delete it if we are confident (can always recover from git) **Files touched**: `components/navbar.tsx`, `components/footer.tsx`, `components/track-switcher.tsx` --- ## Phase 3: New Homepage **Goal**: Replace single-page hero+strengths+showcase with Hero → 4 direction cards → featured work. **Deployable**: Yes — new homepage is live. ### 3.1 `components/direction-card.tsx` — NEW FILE - Large card component for the 2×2 grid on homepage - Props: `{ track: Track, title: string, subtitle: string, description: string, href: string, gradient: string }` - Styling: `bg-white/5 rounded-2xl p-8 md:p-10` — no border - Hover: `hover:scale-[1.02] hover:bg-white/[0.08] transition-all duration-300` - Each card has its own gradient background (subtle): - hydro: `bg-gradient-to-br from-blue-500/10 to-transparent` - ai: `bg-gradient-to-br from-purple-500/10 to-transparent` - devtools: `bg-gradient-to-br from-green-500/10 to-transparent` - indie: `bg-gradient-to-br from-amber-500/10 to-transparent` - Card content: Track label, short description, arrow icon - NOT affected by `data-track` — each card always shows its own color - Links to `/hydro`, `/ai`, `/devtools`, `/indie` - Fade-in + translateY animation with stagger ### 3.2 `components/hero-section.tsx` — Complete rewrite - Remove all track-specific hero variants (HeroHydro, HeroAI, HeroDevtools, HeroIndie) - New homepage hero: full-screen, centered, pure text - Name: `text-5xl md:text-7xl font-bold text-white` (no orbitron) - Tagline: `text-xl md:text-2xl text-white/60 mt-4` - Simple `opacity-0 animate-[fadeIn_0.8s_ease_forwards]` - No buttons, no gradient background — just `bg-[#0A0A0A]` - Takes full viewport height: `min-h-screen flex items-center justify-center` - Export as default for homepage use - Keep the `RichText` helper (useful for descriptions) ### 3.3 `app/page.tsx` — Complete rewrite - Remove imports: TrackStrengths, TrackLatestUpdates, TrackShowcase - New structure: ``` <Navbar /> <HeroSection /> {/* full-screen hero */} <section py-24> {/* 2×2 direction cards */} <DirectionCardGrid /> </section> <section py-24> {/* featured work - optional */} <FeaturedProjects /> </section> <Footer /> ``` - Homepage must NOT set any data-track — it is colorless - Featured work section: show 3-4 highlighted projects across all tracks (use `getAllProjects()` filtered by `featured: true`) ### 3.4 Components to deprecate (no longer used on homepage) - `components/track-strengths.tsx` — no longer on homepage (keep file, used nowhere else → delete later) - `components/track-showcase.tsx` — no longer on homepage (keep file → delete later) - `components/track-latest-updates.tsx` — no longer on homepage (keep file → delete later) **Files touched**: NEW `components/direction-card.tsx`, `components/hero-section.tsx`, `app/page.tsx` --- ## Phase 4: Direction Sub-Pages **Goal**: Create /hydro, /ai, /devtools, /indie as focused showcase pages. **Deployable**: Yes — direction pages are live, clicking homepage cards works. ### 4.1 `components/direction-hero.tsx` — NEW FILE - Reusable hero for direction sub-pages - Props: `{ track: Track, name: string, tagline: string, description: string, buttons: ButtonConfig[] }` - Full-height hero with subtle gradient using the track color: - `bg-gradient-to-b from-[trackColor]/5 to-transparent` - Large centered text, track color for tagline - CTA buttons at bottom - Fade-in animation ### 4.2 `app/hydro/page.tsx` — NEW FILE - Server component - **Critical**: Must set track cookie when visited → use a client component wrapper that calls `setTrack('hydro')` on mount - Hero (via DirectionHero) + curated hydro projects (filtered from getAllProjects) + CTA to /projects?track=hydro - Metadata: title, description for SEO ### 4.3 `app/ai/page.tsx` — NEW FILE - Same pattern as hydro, track='ai' - May include AI tech stack section (simplified from current TrackShowcase AI section) ### 4.4 `app/devtools/page.tsx` — NEW FILE - Same pattern, track='devtools' - May include tool repos section ### 4.5 `app/indie/page.tsx` — NEW FILE - Same pattern, track='indie' - May include metrics section ### 4.6 Shared direction page helper — `components/direction-page.tsx` or inline - Since all 4 pages follow the same Hero + Projects + CTA pattern, consider a shared layout component: ```tsx function DirectionPage({ track, hero, projects }) { return ( <> <TrackSetter track={track} /> {/* client component that sets cookie */} <Navbar /> <DirectionHero {...hero} /> <section className="py-24 md:py-32 px-6 md:px-8"> <ProjectGrid projects={projects} /> </section> <section className="py-24 text-center"> <CTA track={track} /> </section> <Footer /> </> ) } ``` ### 4.7 `components/track-setter.tsx` — NEW FILE (client component) - Tiny client component: `useEffect(() => { setTrack(props.track) }, [])` - Sets the cookie so subsequent /blog, /projects visits are pre-filtered - Does NOT call `router.refresh()` (per requirements) **Files touched**: NEW `components/direction-hero.tsx`, NEW `components/track-setter.tsx`, NEW `app/hydro/page.tsx`, NEW `app/ai/page.tsx`, NEW `app/devtools/page.tsx`, NEW `app/indie/page.tsx` --- ## Phase 5: Existing Page Updates **Goal**: Update card styles, page headers, spacing across existing pages. **Deployable**: Yes — all pages match new design language. ### 5.1 `components/card-components.tsx` — Restyle cards - Update `app/about/page.tsx`: same spacing - Update `app/research/page.tsx`: same spacing - Update `app/tools/page.tsx`: same spacing - Update `app/contact/page.tsx`: same spacing ### 5.5 `components/project-filter.tsx` — Restyle filter badges - Use subtle pill styling instead of shadcn Badge defaults **Files touched**: `components/card-components.tsx`, `components/page-header.tsx`, `components/page-sections.tsx`, `components/project-filter.tsx`, all `app/*/page.tsx` files --- ## Phase 6: Cleanup & Polish **Goal**: Remove dead code, verify all pages, update navigation config. **Deployable**: Yes — final clean state. ### 6.1 Navigation config update - `lib/profile-config.ts`: Update `navigationConfig` to potentially include direction pages, or keep as-is (directions are accessed via homepage cards, not nav) - Clean up heroConfig and track overrides (no longer needed in current form since hero is now generic) - Remove `getTrackHeroConfig` if unused - Remove `strengthsTrackConfig` if StrengthsSection is no longer track-variant ### 6.2 Delete deprecated components - `components/track-switcher.tsx` — no longer imported - `components/track-strengths.tsx` — no longer imported - `components/track-showcase.tsx` — no longer imported - `components/track-latest-updates.tsx` — no longer imported ### 6.3 Clean up design tokens - `lib/design-system/tokens.ts` — remove `shadows.glow`, update typography references ### 6.4 Theme provider decision - Either remove `components/theme-provider.tsx` and `next-themes` dependency entirely - Or keep with `forcedTheme="dark"` for zero-effort dark mode - Recommendation: keep ThemeProvider with forcedTheme="dark" — less risk, easy to re-enable later ### 6.5 Verify track persistence flow - Test: Visit /ai → navigate to /blog → blog should show AI-filtered posts - The flow: /ai page mounts TrackSetter → sets cookie → user navigates to /blog → blog page reads from searchParams (server) but client-side TrackProvider reads cookie and updates URL with ?track=ai → page re-renders with filtered content - **Issue**: `getServerTrack()` only reads searchParams, not cookies. On first server render of /blog after visiting /ai, it will return `DEFAULT_TRACK` because there's no `?track=ai` in the URL. - **Fix**: TrackProvider should update the URL search param on init (it already does in `setTrack`), but the initial page load is SSR. Two options: 1. Middleware reads the `career-track` cookie and redirects to `?track=ai` (safe since middleware already handles cookies) 2. Accept that server render shows all posts, client hydration then filters (causes flash) - **Recommendation**: Option 1 — add track redirect to middleware. Since middleware already sanitizes cookies, it can safely read the ASCII-only `career-track` cookie and append `?track=X` to the URL for pages that support it (/blog, /projects). ### 6.6 `middleware.ts` — Add track redirect - After the non-ASCII cookie cleanup, check for `career-track` cookie - If present and the request is to `/blog` or `/projects` (without existing `?track=` param), redirect with `?track=<value>` - This ensures server-side rendering respects the cookie without calling `cookies()` in server components ### 6.7 Final build verification - `pnpm build` must pass - Visual check all pages in `pnpm dev` **Files touched**: `lib/profile-config.ts`, `middleware.ts`, delete 4 deprecated component files --- ## Dependency Graph ``` Phase 1 (CSS/Theme) ─── must come first, everything depends on it │ ├── Phase 2 (Navbar/Footer) ─── can be done right after Phase 1 │ ├── Phase 3 (New Homepage) ─── depends on Phase 1 for styling │ │ │ └── Phase 4 (Direction Pages) ─── depends on Phase 3 for DirectionCard links │ └── Phase 5 (Existing Pages) ─── depends on Phase 1, can parallel with 3/4 │ └── Phase 6 (Cleanup) ─── must come last ``` ## Risk Mitigations 1. **ByteString crash**: Never call `cookies()` server-side. Middleware cookie reading is safe (it handles raw headers). TrackSetter uses client-side `document.cookie`. 2. **CSS hash mismatch on deploy**: Always deploy via `bash deploy.sh` which does a clean build. Each phase should pass `pnpm build` before deploying. 3. **Content breakage**: Content loading functions (`getContent`, `getNestedContent`, `getAllBlogPosts`, `getAllProjects`) are NOT touched in any phase. Only their consumers change. 4. **Track persistence gap**: Between Phase 3 (homepage no longer sets track) and Phase 4 (direction pages set track), the track cookie may go stale. This is acceptable — the cookie retains the last-set value. 5. **SEO**: Changing homepage structure significantly. Keep metadata in `app/page.tsx`. New direction pages need proper metadata. No URL changes for existing pages. ## File Inventory ### New Files (6) - `app/hydro/page.tsx` - `app/ai/page.tsx` - `app/devtools/page.tsx` - `app/indie/page.tsx` - `components/direction-card.tsx` - `components/direction-hero.tsx` - `components/track-setter.tsx` ### Major Rewrites (5) - `app/globals.css` - `app/layout.tsx` - `app/page.tsx` - `components/hero-section.tsx` - `components/navbar.tsx` ### Moderate Changes (7) - `tailwind.config.ts` - `components/footer.tsx` - `components/card-components.tsx` - `components/page-header.tsx` - `components/page-sections.tsx` - `components/project-filter.tsx` - `middleware.ts` ### Minor Changes (6+) - `lib/profile-config.ts` - `lib/design-system/tokens.ts` - `app/blog/page.tsx` (spacing only) - `app/projects/page.tsx` (spacing only) - `app/about/page.tsx` (spacing only) - Other page files (spacing only) ### Deleted (4) - `components/track-switcher.tsx` - `components/track-strengths.tsx` - `components/track-showcase.tsx` - `components/track-latest-updates.tsx` ### Untouched (preserved as-is) - `lib/track.ts` - `lib/track-server.ts` - `lib/track-content.ts` - `components/track-provider.tsx` - `lib/content.ts` - All `content/` markdown files - `deploy.sh`
