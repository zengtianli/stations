全站重构计划：Apple 风格 + 总览入口架构

PLAN · (plan) · 2026-04-14 09:18 · ~/.claude/plans/hashed-snuggling-galaxy.md
# 全站重构计划：Apple 风格 + 总览入口架构 ## Context 现有网站视觉过时（霓虹色、Orbitron 字体、发光效果、赛博朋克风）。TrackSwitcher 药丸按钮在 navbar 下方，信息架构不清晰。需要彻底重构为 Apple 风格极简设计 + 总览入口页架构。 ## 设计决策总结 | 维度 | 决策 | |------|------| | 架构 | 首页总览 → 4 个方向子页面（/hydro /ai /devtools /indie）| | 主题 | 纯暗色，去掉亮色模式 | | 字体 | 只留 Inter，砍掉 Orbitron/Montserrat/Exo2 | | 首页 | 全屏 Hero（纯文字 fade-in）→ 2×2 渐变大卡片 → 精选作品 | | 方向子页 | Hero + 精选项目(3-5) + CTA | | Navbar | 统一，进入方向后底部一条细色线 | | 配色 | 纯黑 #0A0A0A，降饱和方向色 | | 卡片 | 无边框，bg-white/5 rounded-2xl，hover: scale-[1.02] | | 间距 | py-24 md:py-32 sections, px-6 md:px-8 | | 动画 | 只有 fade-in + translateY | --- ## Phase 1: CSS 基础 + 字体 + 主题 > 所有后续 Phase 依赖此阶段 ### 修改文件 **`app/globals.css`** — 全面重写 - `:root` 背景改为纯黑 `0 0% 4%`（≈#0A0A0A） - 去掉 `--track-glow`, `--track-hero-from`, `--track-hero-to` - 方向色改为降饱和：hydro `217 91% 68%`, ai `263 70% 76%`, devtools `160 64% 52%`, indie `45 93% 56%` - 删除 `.hero-gradient`, `.card-hover` glow, `.timeline-dot` glow, `.typing-effect`, typing/blink keyframes - 去掉 h1 的 `font-orbitron`，h2-h6 的 `font-montserrat` - `--radius` 改为 `1rem` - 保留 `fadeIn`, `fadeInUp` keyframes **`tailwind.config.ts`** — 精简 - 删除 `fontFamily.montserrat`, `.exo2`, `.orbitron` - 更新 radius **`app/layout.tsx`** — 精简字体 - 删除 Montserrat, Exo_2, Orbitron 导入 - ThemeProvider 强制 dark（`forcedTheme="dark"`） - body class 只留 Inter ### 验证 - `pnpm build` 通过 - 现有页面在新 CSS 下渲染正常（可能不完美但不报错） --- ## Phase 2: Navbar + Footer ### 修改文件 **`components/navbar.tsx`** — 重做 - 删除 TrackSwitcher 导入和渲染 - 删除 `font-orbitron` 品牌字体 - 新增：底部 2px 色线，由 `useTrack()` 驱动 - 首页(`/`)不显示 - 方向页/已设方向时显示对应颜色 - 更大的间距，更细的 backdrop-blur **`components/footer.tsx`** — 视觉刷新 - 去掉 `bg-secondary/50`，改为 `border-t border-white/5` - 文字色改为 `text-white/60` ### 验证 - Navbar 干净，无 TrackSwitcher - 访问 `?track=ai` 后 navbar 底部显示紫色线 --- ## Phase 3: 新首页 ### 新建文件 **`components/direction-card.tsx`** — 2×2 大卡片组件 - 各自独立渐变（不受 data-track 影响） - `bg-white/[0.03] rounded-2xl border border-white/[0.06]` - hover: `scale-[1.02]` + 渐变加深 - 链接到 /hydro, /ai, /devtools, /indie - 每张卡片：方向名 + 一句话描述 + `→` 箭头 ### 修改文件 **`components/hero-section.tsx`** — 全面重写 - 删除所有 4 个 track 变体（HeroHydro/HeroAI/HeroDevtools/HeroIndie） - 新 Hero：全屏居中，名字 `text-6xl md:text-8xl font-bold` - 副标题 `text-xl md:text-2xl text-white/50 font-light` - 只有 fade-in 动画 - 不读 useTrack()，首页无色彩 **`app/page.tsx`** — 全面重写 - 结构：Navbar → Hero（全屏）→ 4 卡片区（py-24 md:py-32）→ 精选作品区 → Footer - 精选作品：跨方向取 featured=true 的项目，用新卡片样式 - 不设任何 track ### 验证 - 首页纯黑白，4 张渐变卡片可点击 - 精选作品区展示跨方向项目 --- ## Phase 4: 方向子页面 ### 新建文件 **`components/track-setter.tsx`** — 客户端组件 - mount 时调 `setTrack(props.track)`（写 cookie + 设 data-track） - 不调 router.refresh() **`components/direction-hero.tsx`** — 方向页 Hero - 接收 track 参数，用微妙的方向色渐变（`from-[color]/5 to-transparent`） - 大标题 + 方向色副标题 + CTA 按钮 - 数据来自 `getTrackHeroConfig(track)` **`app/hydro/page.tsx`**、**`app/ai/page.tsx`**、**`app/devtools/page.tsx`**、**`app/indie/page.tsx`** - 统一模式： 1. TrackSetter 设 cookie 2. DirectionHero 3. 精选项目（该方向 featured/highlight 的 3-5 个） 4. CTA（联系/查看全部项目） 5. Navbar + Footer ### 验证 - 点首页「AI 工程」卡片 → 进入 /ai → 看到紫色 Hero + AI 项目 - 然后去 /blog → 自动过滤 AI 博客 --- ## Phase 5: 通用页面视觉升级 ### 修改文件 **`components/card-components.tsx`** - BlogCard/ProjectGrid：去掉 `border-secondary`，用 `bg-white/5 rounded-2xl border-0` - hover 改为 `scale-[1.02]` + `bg-white/[0.08]` **`components/page-header.tsx`** - 标题改大 `text-4xl md:text-5xl` - 间距 `pt-32 pb-16` **`components/page-sections.tsx`** - StrengthsSection 去掉 `bg-secondary/30` - 卡片改 `bg-white/5 rounded-2xl` **`components/research-sections.tsx`** - 卡片去掉 border-secondary，用 `bg-white/5 rounded-2xl` **`components/tool-card.tsx`** - 同上卡片样式更新 **所有 app/*/page.tsx** - 间距统一改为 `py-24 md:py-32 px-6 md:px-8` ### 验证 - 所有通用页面视觉统一 - 卡片无边框、大圆角、微妙 hover --- ## Phase 6: 清理 + Middleware Track 转发 ### 修改文件 **`middleware.ts`** — 加 track cookie 转发 - 读 `career-track` cookie（middleware 读 raw header 安全，不用 cookies()） - 如果请求 /blog 或 /projects 没有 `?track=` 参数，且 cookie 有值，redirect 加上 `?track=` - 解决 SSR 首渲染的 track 不匹配问题 ### 删除文件 - `components/track-switcher.tsx` — 不再引用 - `components/track-strengths.tsx` — 首页已重做 - `components/track-showcase.tsx` — 首页已重做 - `components/track-latest-updates.tsx` — 首页已重做 ### 验证 - `pnpm build` 通过 - 全站 4 方向切换流畅 - 从 /ai 去 /blog 自动带 `?track=ai` --- ## 关键文件清单 | 文件 | 操作 | Phase | |------|------|-------| | `app/globals.css` | 重写 | 1 | | `tailwind.config.ts` | 修改 | 1 | | `app/layout.tsx` | 修改（字体/主题） | 1 | | `components/navbar.tsx` | 重做 | 2 | | `components/footer.tsx` | 刷新 | 2 | | `components/direction-card.tsx` | 新建 | 3 | | `components/hero-section.tsx` | 重写 | 3 | | `app/page.tsx` | 重写 | 3 | | `components/track-setter.tsx` | 新建 | 4 | | `components/direction-hero.tsx` | 新建 | 4 | | `app/hydro/page.tsx` | 新建 | 4 | | `app/ai/page.tsx` | 新建 | 4 | | `app/devtools/page.tsx` | 新建 | 4 | | `app/indie/page.tsx` | 新建 | 4 | | `components/card-components.tsx` | 修改 | 5 | | `components/page-header.tsx` | 修改 | 5 | | `components/page-sections.tsx` | 修改 | 5 | | `components/research-sections.tsx` | 修改 | 5 | | `components/tool-card.tsx` | 修改 | 5 | | `middleware.ts` | 修改 | 6 | | `components/track-switcher.tsx` | 删除 | 6 | | `components/track-strengths.tsx` | 删除 | 6 | | `components/track-showcase.tsx` | 删除 | 6 | | `components/track-latest-updates.tsx` | 删除 | 6 | ## 风险与对策 1. **Phase 2-4 间隙期**：删了 TrackSwitcher 但还没建方向子页时，用户无法切换方向。Cookie 保留上次值，影响有限 2. **ByteString 崩溃**：全程不用 `cookies()` server-side，middleware 读 raw header 安全 3. **内容加载不动**：lib/content.ts 的函数完全不改，只改 UI 消费层 4. **SSR track 不匹配**：Phase 6 middleware 解决，之前可能有短暂闪烁 ## 每个 Phase 后部署验证 ```bash pnpm build && bash deploy.sh # 线上检查：tianlizeng.cloud ```
