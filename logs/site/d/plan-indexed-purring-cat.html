hydro-toolkit 插件集成架构

PLAN · (plan) · 2026-04-01 15:20 · ~/.claude/plans/indexed-purring-cat.md
# hydro-toolkit 插件集成架构 ## Context hydro-apps 有 8 个独立 Tauri app + 1 个空壳 toolkit app。目标：toolkit 编译时集成所有 8 个工具，用户装一个 app 就能用全部功能。独立 app 继续保留。 **UX**: 首页卡片网格 → 点击进入工具 → 侧边栏+tabs → 返回按钮回首页。 **节奏**: 先搭框架 + 集成 annual（最完整的工具）验证，跑通后再批量迁移。 ## Architecture ### Rust 侧：Cargo path 依赖 toolkit 通过 Cargo path 依赖复用每个 app 的核心逻辑，不复制代码。 **Step 1**: 让 annual 的模块可被外部使用 ```rust // apps/annual/src-tauri/src/lib.rs pub mod calc; // 改 mod → pub mod pub mod commands; // 改 mod → pub mod pub mod sample_data; // 改 mod → pub mod ``` **Step 2**: toolkit 依赖 annual crate ```toml # apps/toolkit/src-tauri/Cargo.toml [dependencies] hydro-annual-app = { path = "../annual/src-tauri" } ``` **Step 3**: toolkit 注册 annual 的命令 ```rust // apps/toolkit/src-tauri/src/lib.rs .invoke_handler(tauri::generate_handler![ hydro_annual_app::commands::io::get_sample_data, hydro_annual_app::commands::io::load_data_dir, hydro_annual_app::commands::io::get_indicators, hydro_annual_app::commands::io::export_excel, hydro_annual_app::commands::calc::run_query, hydro_annual_app::commands::calc::run_aggregate, hydro_annual_app::commands::calc::run_compare, hydro_annual_app::commands::calc::run_stats, ]) ``` > **命名冲突处理**：当集成第 2 个 app 时，`export_excel` 等通用名会冲突。届时在 toolkit 里写 thin wrapper 加前缀：`annual_export_excel` / `capacity_export_excel`。MVP 阶段（只有 annual）无冲突，先不加前缀。 ### Frontend 侧：工具路由 + lazy load **新增 `packages/shared` 组件**： 1. **`ToolCard.tsx`** — 首页卡片（icon + 名称 + 描述） 2. **`ToolkitShell.tsx`** — 新的顶层 shell，管理 home/tool 两个视图 ``` toolkit/src/ ├── App.tsx # ToolkitShell + 路由逻辑 ├── Home.tsx # 卡片网格首页 ├── tools/ │ └── annual/ # annual 的前端代码（从 apps/annual/src/ 引用） │ ├── AnnualTool.tsx # 组装 sidebar + tabs（改编自 annual/App.tsx） │ ├── hooks/useAnnual.ts │ ├── pages/QueryPage.tsx │ ├── pages/AggregatePage.tsx │ ├── pages/ComparePage.tsx │ └── lib/commands.ts, types.ts └── main.tsx ``` **App.tsx 核心逻辑**： ```tsx function App() { const [activeTool, setActiveTool] = useState<string | null>(null); if (!activeTool) { return <Home onSelect={setActiveTool} />; } const Tool = TOOLS[activeTool].component; return ( <div style={{ display: "flex", height: "100vh" }}> <aside> <button onClick={() => setActiveTool(null)}>← 返回工具箱</button> <Tool.Sidebar /> </aside> <main> <Tool.Content /> </main> </div> ); } ``` **Home.tsx**：8 个卡片网格，每个卡片含 icon、中文名、英文副标题、一行描述。 ### 前端代码复用策略 为了不复制代码，toolkit 的 tools/annual/ 通过 **TypeScript path alias** 引用 annual 原始源码： ```json // apps/toolkit/tsconfig.json { "compilerOptions": { "paths": { "@tools/annual/*": ["../annual/src/*"] } } } ``` ```ts // apps/toolkit/vite.config.ts resolve: { alias: { "@tools/annual": path.resolve(__dirname, "../annual/src"), } } ``` 这样 toolkit 直接 import annual 的组件，零代码复制。 但 annual 的 `App.tsx` 是独立 app 的入口（包含 AppShell），不适合直接复用。需要抽取一个 **`AnnualPlugin`** 导出 sidebar + tabs： ```tsx // apps/annual/src/plugin.ts (新增) export function useAnnualPlugin() { const annual = useAnnual(); return { sidebar: <AnnualSidebar {...annual} />, tabs: [ { label: "数据查询", content: <QueryPage {...annual} /> }, { label: "汇总分析", content: <AggregatePage {...annual} /> }, { label: "年度对比", content: <ComparePage {...annual} /> }, ], }; } ``` annual 的独立 App.tsx 继续用 AppShell 包装这个 plugin，toolkit 的 App.tsx 也用这个 plugin。**一份逻辑，两种壳。** ## 实现步骤（MVP: toolkit + annual） ### 1. Rust: 开放 annual 模块 + toolkit 依赖 - `apps/annual/src-tauri/src/lib.rs` — `mod` → `pub mod`（3 行改动） - `apps/toolkit/src-tauri/Cargo.toml` — 添加 path 依赖 - `apps/toolkit/src-tauri/src/lib.rs` — 注册 annual 的 8 个 commands ### 2. Frontend: annual 提取 plugin 接口 - `apps/annual/src/plugin.tsx` — 新增，导出 useAnnualPlugin() - `apps/annual/src/AnnualSidebar.tsx` — 从 App.tsx 提取 sidebar 部分 - `apps/annual/src/App.tsx` — 改为使用 plugin.tsx（保持行为不变） ### 3. Frontend: toolkit 首页 + 路由 - `packages/shared/src/components/ToolCard.tsx` — 新增卡片组件 - `apps/toolkit/src/Home.tsx` — 8 工具卡片网格 - `apps/toolkit/src/App.tsx` — home/tool 路由 + 返回按钮 ### 4. Frontend: toolkit 集成 annual - `apps/toolkit/vite.config.ts` — 加 path alias - `apps/toolkit/tsconfig.json` — 加 paths - `apps/toolkit/src/tools/annual.tsx` — thin wrapper 调用 useAnnualPlugin ### 5. 构建验证 ```bash cd apps/toolkit pnpm tauri build # 验证：打开 app → 看到首页 8 个卡片 → 点击"水资源年报" → 使用示例数据 → 查询 → 返回首页 ``` ## 涉及文件 | 文件 | 改动 | |------|------| | `apps/annual/src-tauri/src/lib.rs` | `mod` → `pub mod` | | `apps/annual/src/plugin.tsx` | 新增 | | `apps/annual/src/AnnualSidebar.tsx` | 新增（从 App.tsx 提取） | | `apps/annual/src/App.tsx` | 改用 plugin | | `apps/toolkit/src-tauri/Cargo.toml` | 加依赖 | | `apps/toolkit/src-tauri/src/lib.rs` | 注册 commands | | `apps/toolkit/src/App.tsx` | 重写（路由） | | `apps/toolkit/src/Home.tsx` | 新增 | | `apps/toolkit/src/tools/annual.tsx` | 新增 | | `apps/toolkit/vite.config.ts` | 加 alias | | `apps/toolkit/tsconfig.json` | 加 paths | | `packages/shared/src/components/ToolCard.tsx` | 新增 | | `packages/shared/src/index.ts` | 导出 ToolCard | ## 验证方式 1. `cd apps/annual && pnpm tauri dev` — 确认 annual 独立运行不受影响 2. `cd apps/toolkit && pnpm tauri dev` — 确认 toolkit 能加载 annual 工具 3. toolkit 中：首页 → 点击水资源年报 → 使用示例数据 → 查询正常 → 导出正常 → 返回首页
