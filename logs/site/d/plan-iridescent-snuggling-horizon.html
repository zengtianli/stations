# /services 页面全面升级 + 主题色 Bug 修复 ## Context 服务描述已修正并部署（25 个服务的名称和描述已逐个实证核实）。但还有四类问题： 1. 服务页面内容需要再调整（删三省六部、修复 auggie、删 img） 2. 页面需要升级为完整看板（实时状态、响应时间、搜索过滤） 3. `/services` 未加入导航栏 4. `/blog/[slug]` 和 `/projects/[slug]` 存在主题色 Bug --- ## Phase 1: 服务列表内容修正 ### 1.1 删除三省六部 - `app/services/page.tsx`: 从 SERVICE_GROUPS 删除 board 条目 - VPS: `systemctl stop edict-dashboard && systemctl disable edict-dashboard` - VPS: 删除 `/etc/nginx/sites-enabled/board.tianlizeng.cloud`，reload nginx - CF DNS: `python3 ~/Dev/devtools/lib/tools/cf_api.py dns delete board` - 可选: VPS 上 `/opt/edict` 目录归档或删除 ### 1.2 修复 auggie - 检查 auggie DNS 记录类型（可能是 CNAME 到已失效的外部服务） - 将 DNS 改为 A 记录指向 VPS IP - 在 VPS 部署 auggie-dashboard（Streamlit app，源码在 `~/Dev/auggie-dashboard/`） - 创建 systemd service（streamlit run app.py --server.port 85xx） - 创建 nginx site config - 验证 auggie.tianlizeng.cloud 可访问 ### 1.3 删除 img - `app/services/page.tsx`: 确认已移除（上轮已删） - CF DNS: `python3 ~/Dev/devtools/lib/tools/cf_api.py dns delete img` ### 1.4 cc 分组调整 - 保持"投资工具"独立分组（用户说"都可以"） - 修正 VPS 上 cc-chat.service 的 ExecStart 路径（当前指向已删文件 `/var/www/cc_chat_server.py`，实际跑的是 `/var/www/rh-dashboard/cc-dashboard/app.py`） --- ## Phase 2: 主题色 Bug 修复 **问题**: middleware.ts 用 `pathname === p` 精确匹配，导致 `/blog/some-slug` 和 `/projects/some-slug` 不会被注入 track searchParam，页面背景色默认 hydro 而非用户选择的方向色。 ### 2.1 修复 middleware.ts ```typescript // 改 pathname === p 为前缀匹配 const isTrackFilteredPath = TRACK_FILTERED_PATHS.some( p => pathname === p || pathname.startsWith(p + '/') ) ``` ### 2.2 修复 /blog/[slug]/page.tsx - 导入 `getServerTrack` + `TRACK_PAGE_BG` - 接收 `searchParams` prop - `<main>` 添加 `TRACK_PAGE_BG[track]` ### 2.3 修复 /projects/[slug]/page.tsx - 同上模式 ### 关键文件 - `middleware.ts` ~line 43 - `app/blog/[slug]/page.tsx` - `app/projects/[slug]/page.tsx` --- ## Phase 3: 导航栏 ### 3.1 添加 /services 到导航 - `lib/profile-config.ts`: 在 `navigationConfig` 数组添加 `{ name: "服务", path: "/services" }` - 放在最后一项"生活"之后 --- ## Phase 4: 服务页面升级为完整看板 ### 4.1 后端 — API Route `/api/service-status` - 创建 `app/api/service-status/route.ts` - 从 VPS 本机检查各服务端口（Next.js standalone 跑在 VPS 上，可直接 localhost 探测） - 服务配置数据结构: ```typescript interface ServiceCheck { subdomain: string port: number // VPS 内部端口 type: 'http' | 'tcp' } ``` - 对每个服务发 HTTP GET 到 `http://127.0.0.1:{port}`，记录: - status: 'up' | 'down' - responseTime: number (ms) - checkedAt: ISO string - 内存缓存 60 秒，避免每次请求都探测 - 返回 JSON: `{ services: Record<string, { status, responseTime, checkedAt }> }` ### 4.2 前端 — 服务卡片增强 - `app/services/page.tsx` 改为混合架构: - 服务列表仍然 SSR（SEO） - 新增 `'use client'` 组件 `ServiceStatusDashboard` 负责动态部分 - Service 数据结构扩展: ```typescript interface Service { name: string subdomain: string description: string url?: string port?: number // 用于状态检查 accessType?: 'public' | 'cf-access' // CF Access 标记 } ``` ### 4.3 前端 — UI 元素 **状态指示灯**: 每个服务卡片左侧加圆点 - 绿色脉冲点 = online - 红色 = offline - 灰色 = 未检测（loading / 无端口配置） **CF Access 标记**: 锁图标 🔒（`accessType: 'cf-access'`） **响应时间**: 卡片右侧小字显示 `42ms` **统计栏**: 页面顶部 - X 在线 / Y 离线 / Z 总数 - 替换当前的 "1 台 VPS" 静态文字 **搜索过滤**: 统计栏下方加搜索框 - 即时过滤服务名称和描述 - 可选: 按分组筛选 **自动刷新**: 每 60 秒重新 fetch `/api/service-status` ### 关键文件 - 新建: `app/api/service-status/route.ts` - 新建: `components/service-status.tsx`（客户端组件） - 修改: `app/services/page.tsx` --- ## Phase 5: VPS 清理 ### 5.1 修复 cc-chat.service ```bash # 更新 ExecStart 指向实际运行的文件 sudo systemctl edit cc-chat --full # ExecStart=/usr/bin/python3 -m streamlit run /var/www/rh-dashboard/cc-dashboard/app.py --server.port 8521 --server.headless true sudo systemctl daemon-reload && sudo systemctl restart cc-chat ``` ### 5.2 清理三省六部 ```bash systemctl stop edict-dashboard systemctl disable edict-dashboard rm /etc/nginx/sites-enabled/board.tianlizeng.cloud nginx -t && systemctl reload nginx # /opt/edict 保留但不再运行 ``` ### 5.3 删除 img DNS 记录 ```bash python3 ~/Dev/devtools/lib/tools/cf_api.py dns delete img ``` --- ## 验证清单 1. **服务列表**: 页面无三省六部，auggie 可访问，img 已删 2. **导航栏**: /services 出现在 navbar 最后 3. **主题色**: - 设置 track=ai cookie → 访问 /blog/[slug] → 背景应为 #F0E8FF（紫） - 设置 track=devtools cookie → 访问 /projects/[slug] → 背景应为 #DFFBE9（绿） 4. **实时状态**: /services 页面显示绿/红状态点 + 响应时间 5. **搜索**: 输入关键词可过滤服务 6. **VPS**: `systemctl status cc-chat` 正常，edict-dashboard 已停止 7. **DNS**: `dig auggie.tianlizeng.cloud` 指向 VPS，`dig img.tianlizeng.cloud` 无记录 8. **部署**: `pnpm build` 通过 → `bash deploy.sh` → 线上验证
