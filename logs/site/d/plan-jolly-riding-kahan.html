# ~/Dev 项目展示标准统一计划 ## Context GitHub 上各项目展示质量不一致。以 DocKit 为金标准模板，统一所有项目的 README 格式、badges、截图、双语文档。 现有工具：`~/Dev/scripts/screenshot_repos.py`（Playwright 截图，支持有在线 demo 的 Streamlit 项目）。 ## 金标准模板（DocKit 格式） ```markdown # 项目名 **English** | [中文](README_CN.md) 一句话描述。 **设计哲学/slogan。**（可选） [![Live Demo](https://img.shields.io/badge/Live_Demo-xxx.tianlizeng.cloud-blue?style=for-the-badge)](url) ← 有在线服务时 [![License: MIT](https://img.shields.io/badge/License-MIT-green?style=for-the-badge)](LICENSE) [![Python 3.x+](https://img.shields.io/badge/Python-3.x+-yellow?style=for-the-badge)](https://python.org) --- ### 快速体验 / Try it now（有 demo 时） ![demo](docs/screenshots/demo.png) --- ## Features / 能做什么？（表格） ## Install / 安装 ## Quick Start / 快速上手（代码示例） ## 更多细节（可选） ## License ``` ### 必备要素 | 要素 | 说明 | |------|------| | 语言选择器 | `**English** \| [中文](README_CN.md)` 在标题下第一行 | | Badges | `for-the-badge` 风格，License + Python 版本 + Demo（如有） | | 分隔线 | badges 后、截图后各一条 `---` | | 截图 | `docs/screenshots/demo.png`，1280x800 | | Feature Table | 核心功能列表 | | Install + Quick Start | 安装命令 + 代码示例 | | README_CN.md | 完整中文镜像，结构对称 | --- ## 执行计划 ### Step 1: 重写 cc-harness README（EN + CN） 当前问题：无语言选择器、无 badges、无分隔线、无截图 改动： - 添加语言选择器 - 添加 badges（License: MIT `green`, Python 3.8+ `yellow`），`for-the-badge` 风格 - 添加分隔线 - 调整段落结构对齐模板 - README_CN.md 同步改造 **文件：** - `/Users/tianli/Dev/cc-harness/README.md` - `/Users/tianli/Dev/cc-harness/README_CN.md` ### Step 2: 重写 cc-context README（EN + CN） 当前问题：无语言选择器、badge 风格不对（用的小 badge 不是 for-the-badge）、无分隔线、无截图 改动： - 添加语言选择器 - Badge 改为 `for-the-badge` 风格 - 添加分隔线 - README_CN.md 同步改造 **文件：** - `/Users/tianli/Dev/cc-context/README.md` - `/Users/tianli/Dev/cc-context/README_CN.md` ### Step 3: CLI 工具截图 cc-harness 和 cc-context 是纯 CLI 工具，没有 Web URL。方案： 1. 在本地运行命令获取终端输出 2. 用 Python（rich + 导出 SVG/PNG 或 Playwright 渲染 HTML）生成截图 3. 具体方式：用 `script` 或直接捕获输出，渲染为 HTML，再用 Playwright 截图为 1280x800 PNG 4. 存到各项目 `docs/screenshots/demo.png` 可以扩展 `screenshot_repos.py` 支持 CLI 模式，或写一个小脚本专门处理。 ### Step 4: 扫描修复其余项目 快速检查 cclog、downloads-organizer、hydro-* 系列的 README 是否已符合模板。重点检查： - 语言选择器是否在正确位置 - Badge 风格是否统一（for-the-badge） - 分隔线是否有 - 截图引用路径是否一致（`docs/screenshots/demo.png`） 如有偏差则修正。 ### Step 5: dockit Python 版本修正 `README.md` 和 `README_CN.md` 中 Python badge 从 3.10+ 改为 3.11+（对齐 pyproject.toml `requires-python = ">=3.11"`）。 ### Step 6: .gitignore 统一（附带） 为所有 Python 项目补齐基础条目，不删除项目特有条目： - 补 `.env`（hydro-* 普遍缺失） - 补 build artifacts（`*.egg-info/`, `dist/`, `build/`） - 补 lint cache（`.ruff_cache/`, `.mypy_cache/`, `.pytest_cache/`） - 补 IDE（`.idea/`, `.vscode/`） - cc-context/cc-harness 补 `venv/`, `.venv/` ### Step 7: 依赖版本修复 - hydro-geocode `requirements.txt`：`pandas` → `pandas>=2.0.0` - hydro-reservoir `requirements.txt`：`pandas` → `pandas>=2.0.0` - hydro-efficiency `requirements.txt`：`pandas>=1.5.0` → `pandas>=2.0.0` - hydro-irrigation `requirements.txt`：`pandas>=1.5.0` → `pandas>=2.0.0` --- ## 暂不处理 | 项目 | 原因 | |------|------| | configs | 私有 dotfiles | | hydro-app-tauri | 实验项目 | | oauth-proxy | 内部服务 | | scripts | 私有脚本库 | | edict | 三省六部主题独特，保持原样 | | dockit-raycast | Node 项目，单独处理 | | LICENSE 批量添加 | 需用户逐个决定 | --- ## 验证 -
