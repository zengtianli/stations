终端 "You have mail" 抑制 + 计划任务路径漂移全面修复

PLAN · (plan) · 2026-04-25 12:05 · ~/.claude/plans/mac-mail-scalable-crane.md
# 终端 "You have mail" 抑制 + 计划任务路径漂移全面修复 ## Context 每次开终端弹 `You have mail.` —— 用户烦了。根因是 macOS `login(1)` 检查 `/var/mail/$USER` 非空就显示。**为什么有 mail**：cron 任务 stdout/stderr 被本地 Postfix 投递到 mail spool。 侦察发现这是 **2026-04-20 `~/Dev` 重构** 的遗留漂移，不止一处： | # | 任务 | 现路径 | 真相 | 状态 | |---|---|---|---|---| | 1 | crontab `/sites-health` (08:00) | `~/Dev/docs/health/` | 目录不存在 | ❌ 每天失败 → 投递到 mail | | 2 | LaunchAgent `weekly-scan` | `~/Dev/repo-dashboard/lib/task_analyzer.py` | 已归档到 `_archive/repo-dashboard-20260419/` | ❌ exit 2 (`launchctl list` 显示) | | 3 | LaunchAgent 其他 5 个 (auto-git-sync / briefing-collect / cclog-index / daily-health / paths-audit) | — | 路径全部 OK | ✅ | 用户偏好（来自 memory）：mac 通知中心 > 终端 mail；不需要 mail 这种被动提醒；任务失败要么 macOS 通知，要么静默写日志，不要堆 spool。 ## 修复方案（4 步一刀切） ### Step 1 — 一次性掐断终端 mail 提醒源头 ```bash touch ~/.hushlogin # macOS login(1) 看到这个就不打印 "You have mail." / "Last login" : > /var/mail/$USER # 清空积压 spool（942B → 0） ``` `~/.hushlogin` 是 macOS 官方机制，**不影响**： - macOS 通知中心（osascript display notification 照常） - 邮件投递本身（spool 还会涨，但不再骚扰登录） ### Step 2 — 修 crontab，从源头不再产 mail 编辑 `crontab -e`： ```cron MAILTO="" # 新增：cron 任务输出不再投递本地 mail（关键） 0 8 * * * /Users/tianli/.nvm/versions/node/v22.21.1/bin/claude -p "/sites-health" --allowedTools Bash,Write,Read >> /Users/tianli/Dev/stations/docs/health/site-health-$(date +\%F).md 2>&1 # ^^^^^^^^^^ 改：docs → stations/docs ``` 并预创目录： ```bash mkdir -p ~/Dev/stations/docs/health ``` 双重保险：`MAILTO=""` 让 cron 任何任务的输出都不再投递；路径修对让 `/sites-health` 真能写日志。 ### Step 3 — 删除 weekly-scan LaunchAgent（用户已拍板） ```bash launchctl unload ~/Library/LaunchAgents/com.tianli.weekly-scan.plist rm ~/Library/LaunchAgents/com.tianli.weekly-scan.plist rm -f /tmp/weekly-scan.log /tmp/weekly-scan.err # 顺手清残留 ``` 理由：dashboard 业务已被 ops-console 接手，task_analyzer.py 在归档里已无消费者。 ### Step 4 — 验证 ```bash # 1. 终端不再骚扰 exec zsh -l && echo "✅ 重新登录无 'You have mail'" # 2. cron 改对 + 不再投 mail crontab -l | grep -E "MAILTO|stations/docs" ls -d ~/Dev/stations/docs/health # 3. mail spool 不再涨（隔天再看一次） ls -la /var/mail/$USER # 4. LaunchAgent 状态 launchctl list | grep com.tianli # weekly-scan 不该再 exit 2 # 5. 下次 cron 触发后日志真写到新路径 ls ~/Dev/stations/docs/health/ ``` ## 关键文件 | 文件 | 改动 | |---|---| | `~/.hushlogin` | 新建（空文件） | | `crontab` (用户级) | 加 `MAILTO=""` 头 + 改 `~/Dev/docs/health/` → `~/Dev/stations/docs/health/` | | `~/Dev/stations/docs/health/` | mkdir -p | | `/var/mail/tianli` | 清空 | | `~/Library/LaunchAgents/com.tianli.weekly-scan.plist` | unload + 删除 | ## 不动的部分（确认无漂移） - `briefing-collect` / `auto-git-sync` / `cclog-index` / `daily-health` / `paths-audit` — ProgramArguments 路径全部存在 ✅ - 它们的 stderr/stdout 已重定向到 `/tmp/*.log` —— 不会通过 mail 系统骚扰 - `~/Dev/CLAUDE.md` / 全局 `paths.yaml` —— 这次不动（这俩是 SSOT 不是消费者）
