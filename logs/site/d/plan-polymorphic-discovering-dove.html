PLAN · (plan) · 2026-04-24 15:38 · ~/.claude/plans/polymorphic-discovering-dove.md
# Plan · 全省乡镇/街道行政区划主表 ## Context 当前 `db_reservoirs.db` 只有 `reservoirs.township`（自由文本，仅覆盖**有水库的**乡镇，全省 821 个 distinct 值，分布 69 个县）。不是行政区划 SSOT。 **用户诉求**：构建全省乡镇/街道 DB，对齐 counties 颗粒度，作为第四级（省 → 市 → 县 → 乡镇/街道）权威主表。 **关键发现**：无需上网爬数据。本地 `gis/行政边界乡镇/行政境界（乡镇）.shp` 已是权威 SSOT： - **1335 features**（覆盖全省全部乡镇/街道/乡） - 权威字段：`NAME` / `PNCODE`(4位) / `GNID`(9位) / `province` / `CITY` / `COUNTY` - 已算好：`centerx/centery`（质心）、`labelx/labely`（标注点）、`minx/miny/maxx/maxy`（bbox） - 附加字段：`pinyin` / `ONCENAME`(曾用名) / `ALIASNAME`(别名) / `KEYWORD`(完整地址) / `updatetime` - 坐标系：CGCS2000 (EPSG:4490)，基本等同 WGS84 - 更新时间：2025-02-19 **已知坑**： 1. `NAME` 含 Unicode PUA 字符（`\ue537石街道` = 生僻字），需保留原字符 + 生成 ASCII fallback（用 pinyin 字段兜底） 2. `divisionty` 统一标"四级行政区域（乡）"，**不区分镇/乡/街道**，type 要从 `NAME` 后缀解析 ## 目标产出 1. **新建 `data/db_divisions.db`**（独立 DB，职责分明，未来可加村/社区级不污染水利 DB） 2. 初始表：`townships`（1335 条全省主数据）+ `counties`（视子命 3 决定） 3. `bin/import_townships.py` — 从 shapefile 重建 / reconcile townships 表 4. `bin/township_query.py` — 查县下辖乡镇/街道 + 跑 reservoirs 匹配报告 5. `metadata.yml` 加 `db_divisions` 数据库，Datasette Web UI 同时托管 3 个 DB 6. 匹配报告：`reservoirs.township` ↔ `townships.name` 的匹配 / 失配清单（**不自动回填外键**） ## Schema 设计（独立 DB：`data/db_divisions.db`） **跨 DB 查询**：Datasette / sqlite3 都支持 `ATTACH DATABASE`，匹配脚本里： ```python conn = sqlite3.connect("data/db_reservoirs.db") conn.execute("ATTACH DATABASE 'data/db_divisions.db' AS div") # SELECT r.*, t.gn_code FROM reservoirs r LEFT JOIN div.townships t # ON t.county=r.county AND t.name=r.township; ``` ```sql CREATE TABLE townships ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, -- 原始 NAME（保留 PUA 字符） name_ascii TEXT, -- 拼音 fallback（来自 shapefile.pinyin） type TEXT NOT NULL, -- 镇 / 乡 / 街道 / 其他 county TEXT NOT NULL, -- 对齐 counties.name 粒度 city TEXT NOT NULL, province TEXT NOT NULL DEFAULT '浙江省', pn_code TEXT, -- 4位乡镇码 gn_code TEXT, -- 9位国家码 once_name TEXT, -- 曾用名 alias_name TEXT, -- 别名 label_lng REAL, -- 标注点经度（labelx） label_lat REAL, -- 标注点纬度（labely） center_lng REAL, -- 质心经度（centerx） center_lat REAL, -- 质心纬度（centery） bbox_minx REAL, bbox_miny REAL, bbox_maxx REAL, bbox_maxy REAL, source TEXT, -- '行政境界（乡镇）.shp 2025-02-19' source_updated TEXT, -- shapefile.updatetime created_at TEXT DEFAULT (datetime('now')), updated_at TEXT DEFAULT (datetime('now')), UNIQUE(county, name) -- 同县不重名 ); CREATE INDEX idx_townships_county ON townships(county); CREATE INDEX idx_townships_city ON townships(city); CREATE INDEX idx_townships_type ON townships(type); ``` **为什么不存 polygon geometry**：shapefile 本身 150 MB，polygon 进 SQLite 会让 DB 暴涨且 Datasette 渲染不动。查边界仍走 `gis/` 目录的 shapefile（GIS 工具天然支持）。DB 存 bbox + 质心，够做"这个点落在哪个乡镇"的反查 + 地图打点。 ## type 解析规则 按 `NAME` 后缀优先级（长匹配优先）： ``` "街道" -> 街道 "镇" -> 镇 "乡" -> 乡 其他 -> 其他 (极少数，如林场/开发区) ``` ## 导入脚本 `bin/import_townships.py` ``` 用法： python3 bin/import_townships.py reconcile # 从默认 shp 重建 python3 bin/import_townships.py reconcile --dry-run # 预览 python3 bin/import_townships.py reconcile --shp <path> # 指定 shp python3 bin/import_townships.py match # 跑 reservoirs.township 匹配报告 ``` **依赖**：`/opt/homebrew/bin/python3` 已装 `osgeo.ogr`（GDAL），直接可用，不加新依赖。 **reconcile 逻辑**： 1. 打开 shp，遍历 1335 features 2. 对每行抽字段 → upsert by `UNIQUE(county, name)` 3. 打印摘要：新增 / 更新 / 跳过 4. `--dry-run` 只打印 diff，不写库 **match 子命令**（不改数据，只报告）： 1. `SELECT DISTINCT county, township FROM reservoirs WHERE township IS NOT NULL` 2. 对每条按 `(county, township)` 查 townships，报命中率 3. 失配项按"县内同字 / 曾用名 / 别名"模糊匹配建议 4. 输出到 `output/townships-match-report.md`，**不自动写回 reservoirs** ## 对齐 counties 颗粒度 当前 `db_reservoirs.counties` 只有 7 条（项目工作目录映射，非全省 SSOT）。 **本次一并建全省 counties SSOT（在新 DB `db_divisions.db` 里）**： 从 shapefile 聚合 `DISTINCT (province, CITY, COUNTY)` → ~90 条县市。Schema： ```sql CREATE TABLE counties ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL UNIQUE, -- 县名（吴兴区/龙泉市） city TEXT NOT NULL, -- 地级市（湖州市） province TEXT NOT NULL DEFAULT '浙江省', source TEXT, created_at TEXT DEFAULT (datetime('now')) ); ``` `db_reservoirs.counties` 保留原样不动（项目工作目录映射仍有用）。**两表同名但职责不同**： - `db_reservoirs.counties` — 项目承接清单（enabled / project_dir） - `db_divisions.counties` — 全省 SSOT 字符串惯例对齐 `reservoirs.county`（`吴兴区` / `龙泉市` / `德清县`），方便 JOIN。 ## Datasette metadata `metadata.yml` 新增 `db_divisions` 顶级数据库： ```yaml db_divisions: title: 行政区划 description: 浙江省县 / 乡镇 / 街道主数据（SSOT，2025-02 shapefile 版） tables: townships: title: 乡镇/街道 description: 全省 1335 条乡镇 / 乡 / 街道 label_column: name sort: name columns: name: 名称 type: 类型 county: 县（市、区） city: 地级市 pn_code: 乡镇码 gn_code: 国家行政码 center_lng: 质心经度 center_lat: 质心纬度 once_name: 曾用名 counties: title: 县市目录（全省 SSOT） label_column: name columns: name: 县（市、区） city: 地级市 ``` `serve.sh` 更新：`datasette serve` 同时挂载 3 个 DB。`center_lng/center_lat` 自动触发 `datasette-cluster-map`。 ## 验证计划 **写入后**（注意 DB 路径是 `db_divisions.db`）： ```bash # 1. 总数对齐 sqlite3 data/db_divisions.db "SELECT COUNT(*) FROM townships" # 期望 1335 sqlite3 data/db_divisions.db "SELECT COUNT(*) FROM counties" # 期望 ~90 # 2. 吴兴区清单核对（用户给的 5镇+1乡+13街道 = 19 条） sqlite3 data/db_divisions.db \ "SELECT name, type FROM townships WHERE county='吴兴区' ORDER BY type, name" # 期望：19 条 — 织里/八里店/妙西/埭溪/东林 镇 + 道场乡 + 13 街道 # 3. 类型分布全省 sqlite3 data/db_divisions.db "SELECT type, COUNT(*) FROM townships GROUP BY type" # 4. 每县乡镇数 Top 10 sqlite3 data/db_divisions.db \ "SELECT county, COUNT(*) c FROM townships GROUP BY county ORDER BY c DESC LIMIT 10" # 5. 水库匹配率（跨 DB ATTACH） python3 bin/township_query.py match # 期望：> 90% 命中；失配 < 10%（撤并乡镇 / typo） # 6. Datasette Web UI ./bin/serve.sh # 验证：3 个 DB 都挂上；db_divisions > townships 有地图聚类；吴兴区筛出 19 条 ``` ## 明确不做（范围界限） - ❌ 不导 polygon geometry（走 shapefile） - ❌ 不自动回填 `reservoirs.township_id` 外键（先出匹配报告，二期决定） - ❌ 不改 `counties` 表结构 - ❌ 不爬网络数据（本地 shp 已够权威） - ❌ 不做"一个点落在哪个乡镇"的 spatial query（可后续加，走 gis/） ## 关键文件 | 文件 | 动作 | |---|---| | `data/db_divisions.db` | **新建 DB**（首次 reconcile 创建） | | `bin/import_townships.py` | **新建**（按 `import_reservoir.py` 风格） | | `bin/township_query.py` | **新建**：`list <县>` / `match` / `counties` | | `bin/serve.sh` | 加一行挂 `data/db_divisions.db` | | `metadata.yml` | 追加 `db_divisions` 数据库（townships + counties） | | `README.md` | 更新目录结构表 + CLI 示例 | | `CLAUDE.md` | 加「三个数据库分工」+ 跨 DB ATTACH 模式 | | `.claude/commands/query-townships.md` | **新建** slash command | ## 执行顺序 1. 写 `bin/import_townships.py`（含 reconcile + counties 聚合） 2. `reconcile --dry-run` 看摘要（1335 条预览） 3. `reconcile` 正式创建 `data/db_divisions.db` 4. 跑验证 1-4（行数 + 吴兴区 19 条 + 类型分布 + Top 10） 5. 写 `bin/township_query.py` 6. `match` 生成 `output/townships-match-report.md`（不写回 reservoirs） 7. 更新 `serve.sh` / `metadata.yml` / `README.md` / `CLAUDE.md` 8. 新建 `.claude/commands/query-townships.md` 9. `./bin/serve.sh` 肉眼验证 Web UI 10. git commit（`feat: 新增 db_divisions 全省乡镇/街道主数据库`）
