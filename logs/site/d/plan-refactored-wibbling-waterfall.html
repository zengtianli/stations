四个脚本 Repo 迁移到 uv 运行环境

PLAN · (plan) · 2026-04-13 16:49 · ~/.claude/plans/refactored-wibbling-waterfall.md
# 四个脚本 Repo 迁移到 uv 运行环境 ## Context 用户已从 miniforge/pip 迁移到 uv 管理 Python 环境。doctools 刚修了 `run_python.sh` 用 `uv run`，但其余 3 个 repo 和一些遗留文件还在用旧方式（`python3` 直调、miniforge PATH）。需要统一。 ## 现状 | repo | pyproject.toml | .venv | run_python.sh | common.sh miniforge | |------|:-:|:-:|:-:|:-:| | doctools | YES | YES | `uv run` (已修) | YES (残留) | | devtools | YES | YES | `python3` | YES | | mactools | NO | NO | `python3` | 用 `/opt/homebrew/bin` | | clashx | NO | NO | `python3` | YES | 另有 3 个 Raycast wrapper 绕过 run_python.sh 直接 `which python3`： - `doctools/raycast/commands/doc_view_md.sh` - `devtools/raycast/commands/git_smart-push.sh` - `mactools/raycast/commands/file_cleanup-downloads.sh` ## 改动计划 ### 1. run_python.sh — 统一用 `uv run`（3 个文件） 对 **devtools, mactools, clashx** 的 `raycast/lib/run_python.sh`： ``` - exec python3 "$SCRIPTS_DIR/$script_name" "$@" + exec uv run --project "$PROJECT_ROOT" python3 "$SCRIPTS_DIR/$script_name" "$@" ``` mactools 和 clashx 没有 pyproject.toml，`uv run --project` 会报错。需要先为它们创建 pyproject.toml（见第 3 步）。 ### 2. common.sh — 去掉 miniforge 残留，加 uv/homebrew PATH（4 个文件） 4 个 repo 的 `lib/common.sh` 都要改 22-26 行环境变量段： ```bash # 旧（doctools/devtools/clashx） readonly MINIFORGE_BIN="$HOME/miniforge3/bin" if [ -d "$MINIFORGE_BIN" ]; then export PATH="$MINIFORGE_BIN:$PATH" fi # 新（统一） # 确保 uv 和 homebrew 的 bin 在 PATH 中 for _bin_dir in "$HOME/.local/bin" "/opt/homebrew/bin"; do [ -d "$_bin_dir" ] && export PATH="$_bin_dir:$PATH" done ``` mactools 已经用 `/opt/homebrew/bin`，但变量名还叫 `MINIFORGE_BIN`，也要统一。 ### 3. mactools & clashx — 创建 pyproject.toml **mactools/pyproject.toml**：检查脚本 import 需要什么依赖（llm_client 等是本地 lib，不需要额外包）。 **clashx/pyproject.toml**：全是 shell 脚本，只有 `clashx.sh` 里一个 `python3 -c` 内联调用，依赖为空即可。 ### 4. 3 个绕过 run_python.sh 的 Raycast wrapper — 改用 run_python.sh 这 3 个文件手写了路径检测 + `which python3`，应该统一走 `run_python.sh`： **doctools/raycast/commands/doc_view_md.sh**： ```bash source "$(dirname "$(realpath "$0")")/../lib/run_python.sh" && run_python "document/md_tools.py" to-html "$1" ``` **devtools/raycast/commands/git_smart-push.sh**： ```bash source "$(dirname "$(realpath "$0")")/../lib/run_python.sh" && run_python "tools/git_smart_push.py" ``` **mactools/raycast/commands/file_cleanup-downloads.sh**： 这个比较特殊，连续调用 3 个脚本且有错误检查。不适合直接用 `run_python`（因为 exec 会退出）。改法：common.sh 里用 `uv run` 包装 `$PYTHON`，或者在这个文件里直接用 `uv run --project "$REPO_ROOT" python3`。 ### 5. 文件清单 | 文件 | 操作 | |------|------| | `doctools/lib/common.sh` | 改 miniforge → uv+homebrew PATH | | `devtools/lib/common.sh` | 改 miniforge → uv+homebrew PATH | | `devtools/raycast/lib/run_python.sh` | python3 → uv run | | `mactools/lib/common.sh` | 改变量名，加 ~/.local/bin | | `mactools/raycast/lib/run_python.sh` | python3 → uv run | | `mactools/pyproject.toml` | 新建 | | `clashx/lib/common.sh` | 改 miniforge → uv+homebrew PATH | | `clashx/raycast/lib/run_python.sh` | python3 → uv run | | `clashx/pyproject.toml` | 新建 | | `doctools/raycast/commands/doc_view_md.sh` | 改用 run_python.sh | | `devtools/raycast/commands/git_smart-push.sh` | 改用 run_python.sh | | `mactools/raycast/commands/file_cleanup-downloads.sh` | python3 → uv run | ## 验证 1. 每个 repo 下 `uv run python3 -c "print('ok')"` 能通过 2. Raycast 里触发 `docx-text-format` 确认仍能跑 3. 各 repo `git diff` 确认改动合理，commit + push
