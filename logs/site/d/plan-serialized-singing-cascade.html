# Plan: Dashboard 指标重构 — 落实 strategy-roll.md 到代码和页面 ## Context Dashboard 当前的"年化(成本)"和"杠杆年化"只显示**剩余 TV 的 forward-looking 年化**。对于 deep ITM 近到期仓位，这个数字接近 0 或负数，给人"这笔交易亏钱"的错觉。实际上，roll credit（开仓时收到的 net credit）年化可能高达 30%+。 用户要求： 1. 显示多个年化指标，让每笔交易的真实收益一目了然 2. 显示滑点，辅助 roll vs assign 决策 3. 在 Dashboard 页面内加策略说明，新手也能看懂各指标含义 ## 修改文件 1. `scripts/st_snapshot.py` — 保存 bid/ask/spread 到 portfolio.json 2. `scripts/lib_greeks.py` — 新增 roll credit 匹配 + bid/ask 传递 3. `scripts/app.py` — 重构期权表列 + roll 目标改 ATM + 加说明 expander --- ## 1. st_snapshot.py — 保存 bid/ask `_yahoo_mid_prices()` 改为 `_yahoo_option_prices()`，返回 `{key: {"mid": x, "bid": y, "ask": z, "spread": z-y}}`。 写入 portfolio.json 时，除 `o["price"] = mid` 外，追加： ```python o["bid"] = info["bid"] o["ask"] = info["ask"] o["spread"] = info["spread"] ``` ## 2. lib_greeks.py — 传递 bid/ask + 新增 roll credit 匹配 ### process_portfolio() 改动 opt_row 新增字段： ```python "bid": float(o.get("bid") or 0), "ask": float(o.get("ask") or 0), "spread": float(o.get("spread") or 0), ``` ### 新增函数 compute_roll_credits(opt_rows, activities_path) 从 activities.json 匹配每个持仓的开仓交易，计算 roll credit： ```python def compute_roll_credits(opt_rows, activities_path): """Match current positions with opening trades, return {ticker: roll_info}.""" # 1. 加载 activities，筛选 option trades # 2. 对每个 opt_row 的 ticker，找 SELL（type=SELL）记录 # 3. 取最近一条 SELL 作为开仓交易（sell_price, sell_date） # 4. 在同一天找同 strike 不同 expiry 的 BUY（平仓旧仓） # - 如果找到：roll_credit = sell_price - buy_price # - 如果没找到：fresh_open，credit = sell_price（即 avg） # 5. 计算 roll 年化 = (roll_credit / strike) × (365 / total_dte) × 100 # 返回 {ticker: {"credit": x, "annual": y, "open_date": z}} ``` ## 3. app.py — 重构期权表 ### 列变更 删除/重命名： - "年化(成本)" → **"剩余TV年化"**（含义不变，改名更清晰） - "杠杆年化" → 保留 新增 3 列： - **"Roll年化"** — 开仓 roll credit 的年化收益（backward-looking，真实回报） - **"滑点"** — bid-ask spread × 张数 × 100（$ 金额） - Roll 目标列中加 **"→ ATM"**（不再按 delta 推 deep ITM strike） ### 表格列顺序 ``` 到期 | DTE | 行权价 | 数量 | 现价 | IV | Delta | Theta | 剩余TV | TV收割 | 收取权利金 | Roll年化 | 剩余TV年化 | 杠杆年化 | 滑点 | 建议 | Roll目标 | 理由 ``` ### Roll 目标改为 ATM ```python if "ROLL" in roll_action: target_strike = round(data["qqq_price"]) # 永远 ATM target_dte = 30 target_exp = (datetime.now() + timedelta(days=target_dte)).strftime("%m/%d") est_tv = r["iv"] * data["qqq_price"] * math.sqrt(target_dte / 365) * 0.4 # ATM TV estimate est_yield = (est_tv / target_strike) * (365 / target_dte) * 100 roll_target = f"→ {target_strike}C {target_exp} ATM (~{est_yield:.0f}%年化)" ``` ### Roll vs Assign 建议逻辑（来自 strategy-roll.md Section 2） `compute_roll_signals()` 增加 spread 参数，改造建议逻辑： ```python # Deep ITM + 近到期 → 比较滑点 vs 剩余TV if r["delta"] > 0.85 and r["dte"] <= 7: slippage = r["spread"] * abs(r["qty"]) * 100 tv_left = abs(r["qty"]) * r["time_value"] * 100 if slippage > tv_left: action = "LET ASSIGN → 卖ATM" reason = f"滑点${slippage:,.0f} > 剩余TV${tv_left:,.0f}，assign后重开ATM更划算" else: action = "ROLL" # ATM + QQQ 在跌 → 积极 calendar roll（IV高、spread小） # ATM + TV≥80% → calendar roll # Deep ITM + DTE>30 → HOLD（TV还在衰减） # Slight OTM → HOLD（不追跌，不 roll down） ``` 完整决策表（对应 strategy-roll.md Section 2 决策总结）： | 当前状态 | 操作 | 原因 | |---------|------|------| | Deep ITM + DTE ≤ 7 + 滑点 > TV | LET ASSIGN → 卖ATM | 滑点太大 | | Deep ITM + DTE ≤ 7 + 滑点 < TV | ROLL | 滑点可控 | | Deep ITM + DTE > 30 | HOLD | TV 还在衰减 | | ATM + TV ≥ 80% | ROLL (calendar) | 同 strike 延期 | | ATM + QQQ 在跌 + IV高 | ROLL (积极) | IV高→premium肥 | | Slight OTM | HOLD | 不追跌，不 roll down | ### IV 择时显示（来自 strategy-roll.md Section 4） 在建议列或理由列中体现当前 IV 水平对 roll 时机的影响： - IV > 35% → 理由追加"IV高，积极roll" - IV < 20% → 理由追加"IV低，可推迟roll" ### 策略说明 Expander（来自 strategy-roll.md 核心内容） 在期权表下方加 `st.expander("策略说明 & 指标解读")`，渲染 strategy-roll.md 的关键内容： **Part 1: 指标说明** | 指标 | 含义 | 怎么看 | |------|------|--------| | **Roll年化** | 这笔 roll 操作的实际年化收益（开仓 credit / strike × 365/DTE） | 越高说明开仓时机越好。deep ITM 到期前这个值仍然是正的——说明交易赚钱了，不是亏了 | | **剩余TV年化** | 从现在到到期还能赚多少（剩余时间价值年化） | 接近 0 说明 theta 已收割完毕，该 roll 或 let assign | | **杠杆年化** | 剩余TV扣除 margin 利息后的年化 | 负数说明持有成本已超过剩余收入，应立即行动 | | **滑点** | Roll 这张期权需要付出的 bid-ask spread 成本 | 如果 > 剩余TV，let assign + 重开 ATM 比 roll 更划算 | | **TV收割** | 已收割的时间价值占比（= 1 - 剩余TV/开仓premium） | ≥80% 触发 roll 信号 | **Part 2: 策略循环（来自 strategy-roll.md Section 1）** ``` 开仓：卖 ATM Call（delta ~0.50，theta 最大） ↓ QQQ 上涨 变成 ITM（delta 0.80+）→ 自动成为对冲层 ↓ TV 耗尽 Let assign → 重新卖 ATM → 循环 ``` **Part 3: Roll vs Assign 决策（来自 strategy-roll.md Section 2）** - 绝不 roll up / roll down - Deep ITM：let assign + 重开 ATM（省 95% 滑点） - ATM：calendar roll（同 strike 延期） - 滑点 > 剩余TV → let assign 更划算 **Part 4: IV 择时（来自 strategy-roll.md Section 4）** | IV | 操作 | 原因 | |----|------|------| | > 35% | 积极 roll | premium 膨胀 | | 20-35% | 正常 roll | 按 TV 收割率 | | < 20% | 推迟 roll | premium 太便宜 | **Part 5: 实绩对比（来自 strategy-roll.md Section 0）** | 指标 | CC 实盘 | TQQQ 3x | QQQ 1x | |------|--------|---------|--------| | 累计收益(TWR) | +71.6% | +113.1% | +37.0% | | Sharpe | 4.04 | 1.67 | 1.73 | | MaxDD | -11.1% | -37.2% | -11.6% | ## 数据流 ``` st_snapshot.py 拉 Yahoo bid/ask → portfolio.json 新增 bid/ask/spread 字段 ↓ lib_greeks.py process_portfolio() → opt_row 新增 bid/ask/spread lib_greeks.py compute_roll_credits() ← activities.json → {ticker: roll_info} ↓ app.py 合并 opt_row + signal + roll_credit → 渲染表格 ``` ## 验证 1. `python3 scripts/st_snapshot.py` — 确认 portfolio.json 包含 bid/ask/spread 2. `python3 -c "..."` — 验证 roll credit 匹配正确（$568C → credit $7.61） 3. 语法检查 `ast.parse()` 4. 本地 `streamlit run scripts/app.py` 验证渲染 5.
