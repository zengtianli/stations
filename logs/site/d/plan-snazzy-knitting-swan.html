# 数据准备模块自动化：7 模块独立命令 ## Context gen-report 命令已完成（逐章 LLM 生成报告），但报告生成依赖的**数据准备**环节大部分是手动的。现在要把 7 个数据模块逐一自动化，每个模块一个独立的 CC 命令。目标：新县市只需运行几个命令就能准备好所有数据，然后直接 `/gen-report` 生成报告。 ## 模块总览与实施状态 | 模块 | 命令 | 脚本 | 状态 | 说明 | |------|------|------|------|------| | A 基础信息 | `/prep-basic-info` | CC命令直接编排 | **待实现** | 网搜+下载+LLM提取→01-06.md | | B 统计数据 | `/prep-stats` | CC命令+模板 | **待实现** | 从公报数据生成供水/用水/经济Excel | | C 工程排查 | `/prep-engineering` | generate_table_2_1.py | **已有脚本，缺命令** | 一键筛选→合并→提取→简化 | | D 生态流量计算 | `/prep-ecoflow-calc` | eco_flow_calc.py(新) | **待实现** | Python Tennant/QP法计算 | | E 水功能区 | — | — | **暂缓** | 用户后续提供数据源 | | F 参考表格 | — | — | **暂缓** | 用户后续提供数据源 | | G 工程保障 | — | — | **无法自动化** | 依赖现场查勘 | ## 模块 A：/prep-basic-info（基础信息） ### 目标 为目标县市生成 `data/01_基本信息.md` ~ `06_水资源概况.md`，数据来源是网上的水利报告和统计资料。 ### 产出文件 ``` {county}/data/ ├── 01_基本信息.md # 地理位置、坐标、面积、地形地貌 ├── 02_行政区划.md # 乡镇街道数量、行政村数 ├── 03_经济社会.md # 常住人口、GDP、产业结构 ├── 04_河流水系.md # 所属水系、主要河流、流域面积 ├── 05_水文气象.md # 年均气温、降水量、蒸发量 └── 06_水资源概况.md # 水资源总量、供水用水概况 ``` ### 实现方式：CC 命令（非 Python 脚本） 这个模块的核心是**信息检索+LLM提取**，适合用 CC 命令直接编排，利用 Claude 的 WebSearch + WebFetch 能力。 ### 命令文件 `~/Dev/cc-harness/commands/prep-basic-info.md` ### 工作流程 1. 从 `$ARGUMENTS` 解析县市名，读 `sync_config.yaml` 获取 city/stat_name 2. 以景宁的 01-06 MD 为**模板**，了解每个文件需要哪些字段 3. 用 WebSearch 搜索： - `"{stat_name} 水资源公报 2023"` 或 `"{stat_name} 水资源公报 2024"` - `"{stat_name} 统计年鉴"` 或 `"{city} 统计年鉴 {stat_name}"` - `"{stat_name} 河流水系"` / `"{stat_name} 水文气象"` - `"{city} 水利发展统计公报"` 4. 用 WebFetch 下载找到的 PDF/网页内容到 `{county}/data/参考资料/` 5. 读取下载的资料，用 LLM 按景宁模板格式提取结构化数据 6. 生成 6 个 MD 文件到 `{county}/data/` 7. 标记哪些字段是确认的、哪些是待核实的（用 `<!-- 待核实 -->` 注释） ### 参考模板 景宁的 6 个 MD 文件作为格式参考：`~/Work/eco-flow/lishui-景宁/data/01_*.md` ~ `06_*.md` ### 关键约束 - 数据来源必须标注出处（哪份公报、哪个年份） - 找不到的字段标记 `[待补充]`，不编造 - 优先使用最新年份数据（2023/2024） --- ## 模块 B：/prep-stats（统计数据） ### 目标 生成县市的统计数据 Excel：供水量、用水量、经济社会数据。 ### 产出文件 ``` {county}/data/统计数据/ ├── {stat_name}_供水量数据.xlsx # 5年供水量（地表水/地下水/再生水） ├── {stat_name}_用水量数据.xlsx # 5年用水量（农业/工业/生活/生态） └── {stat_name}_经济社会数据.xlsx # 5年人口+GDP ``` ### 实现方式：CC 命令 与模块 A 类似，核心是信息检索。统计数据主要来自： - 各市水资源公报（PDF 表格） - 国家/省统计局网站 - 已同步的数据（sync_project.py 可能已经同步了部分） ### 命令文件 `~/Dev/cc-harness/commands/prep-stats.md` ### 工作流程 1. 检查是否已有（sync_project.py 已同步的统计数据） 2. 若缺失，WebSearch 搜索水资源公报 3. 从公报中提取 5 年供水/用水/经济数据 4. 用 Python 生成标准格式 Excel（pandas） 5. 对照景宁的 Excel 结构确保字段一致 ### Excel 格式参考（景宁） - 供水量：年份 | 地表水(亿m³) | 占比 | 地下水(亿m³) | 占比 | 再生水(亿m³) | 占比 | 总计 - 用水量：年份 | 农业灌溉 | 占比 | 林牧渔畜 | 占比 | 工业 | 占比 | 综合生活 | 占比 | 生态环境 | 占比 | 总计 - 经济社会：年份 | 常住人口(万人) | 地区生产总值(亿元) --- ## 模块 C：/prep-engineering（工程排查） ### 目标 一键生成表 2-1（小型水库基本情况和生态流量状况表）。 ### 状态：✅ 脚本已有，只需包装 CC 命令 ### 已有脚本 `~/Work/eco-flow/code/脚本/core/generate_table_2_1.py` - 流程：筛选 → 合并 → MD 提取 → 字段简化 - 依赖全省数据源 ### 命令文件 `~/Dev/cc-harness/commands/prep-engineering.md` ### 工作流程 1. 解析县市名，读 sync_config.yaml 2. 先运行 `sync_project.py sync {county}` 同步最新数据 3. 运行 `generate_table_2_1.py --county {county} --city {city}` 4. 检查输出：合并数据/水库基本情况和生态流量状况_{county}.xlsx 5. 报告水库数量和数据完整度 --- ## 模块 D：/prep-ecoflow-calc（生态流量计算） ### 目标 用 Python 实现 Tennant 法和 QP 法计算，替代 Windows CurveFitting.exe。 ### 产出文件 ``` {county}/data/计算结果/ ├── {county}_生态流量计算结果.xlsx # 所有水库的 Tennant/QP 计算成果 ├── {county}_逐月流量统计.xlsx # 各水库逐月平均流量 └── 计算说明.md # 方法、参数、数据来源说明 ``` ### 新建脚本 `~/Work/eco-flow/code/脚本/core/eco_flow_calc.py` ### 输入数据 - `~/Work/eco-flow/data/小型水库流量_水文to水资源.xlsx`（47个水库的1962-2023逐日流量） - `~/Work/eco-flow/code/水文频率工具/*.txt`（频率分析原始数据，可选验证） - 合并数据表（获取水库集雨面积等参数） ### 计算方法 **Tennant 法（Montana 法）**： - 以多年平均流量为基准 - 生态基流 = 多年平均流量 × 比例系数 - 丰水期（4-9月）：30%（良好）/ 20%（一般）/ 10%（差） - 枯水期（10-3月）：20%（良好）/ 10%（一般）/ 5%（差） - 推荐取枯水期 10%（一般状况） **QP 法（90% 频率最枯月流量）**： - 统计逐月平均流量 - 对最枯月进行频率分析 - 取 P=90% 的枯水流量作为生态基流 **最终推荐**： - 取 Tennant 法和 QP 法计算结果中的较大值 ### 脚本设计 ```python 用法: python3 eco_flow_calc.py \ --flow-data <逐日流量xlsx> \ --county <县名> \ --output <输出目录> \ [--reservoirs <水库名列表，逗号分隔>] # 默认全部 [--period 1962-2023] # 计算时段 [--dry-run] # 只打印不写文件 ``` 关键函数： - `load_flow_data()` — 读取逐日流量 xlsx，按水库分列 - `calc_monthly_avg()` — 逐月平均流量统计 - `calc_tennant()` — Tennant 法计算（多年平均 × 系数） - `calc_qp()` — QP 法（频率分析，P=90%） - `recommend_ecoflow()` — 综合推荐值 - `export_results()` — 输出 Excel + 说明 MD ### 命令文件 `~/Dev/cc-harness/commands/prep-ecoflow-calc.md` ### 工作流程 1. 解析县市名 2. 从合并数据表获取该县市水库列表 3. 从逐日流量 xlsx 筛选对应水库数据 4. 运行 Tennant + QP 计算 5. 生成结果 Excel 和说明文档 6. 对比已有 CurveFitting 结果验证（如有） --- ## 新增文件清单 | 类型 | 文件路径 | 说明 | |------|---------|------| | CC命令 | `~/Dev/cc-harness/commands/prep-basic-info.md` | 模块A：基础信息网搜+提取 | | CC命令 | `~/Dev/cc-harness/commands/prep-stats.md` | 模块B：统计数据准备 | | CC命令 | `~/Dev/cc-harness/commands/prep-engineering.md` | 模块C：工程排查包装 | | CC命令 | `~/Dev/cc-harness/commands/prep-ecoflow-calc.md` | 模块D：生态流量计算 | | Python | `~/Work/eco-flow/code/脚本/core/eco_flow_calc.py` | Tennant/QP法计算脚本 | ## 复用已有工具 | 工具 | 路径 | 用于模块 | |------|------|---------| | generate_table_2_1.py | `~/Work/eco-flow/code/脚本/core/` | C | | sync_project.py | `~/Work/eco-flow/` | B, C | | llm_client.py | `~/Dev/devtools/scripts/tools/` | A, B | | sync_config.yaml | `~/Work/eco-flow/` | 所有模块 | ## 实施顺序 1. **模块 C**（最简单，脚本已有，只需命令包装） 2. **模块 D**（纯计算，Python 实现明确） 3. **模块 A**（最复杂，网搜+LLM提取） 4. **模块 B**（与 A 类似但数据更结构化） ## 验证方案 - **模块 C**：对景宁跑一次，对比已有 `合并数据/` 输出 - **模块 D**：对景宁的水库跑 Tennant/QP，对比 CurveFitting 结果和 `生态流量核定.xlsx` - **模块 A**：对天台跑一次，人工审核 6 个 MD 文件的准确性 - **模块 B**：对天台跑一次，对比景宁 Excel 格式是否一致 ## 记忆更新 实施前先保存项目记忆：各模块完成状态、数据依赖关系、暂缓模块原因。
