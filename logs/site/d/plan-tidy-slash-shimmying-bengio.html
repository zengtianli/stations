# Streamlit → Next.js 全迁 · Turborepo monorepo · /stack-classify 分类清单 ## Context ### 问题 `~/Dev` 50+ 项目现状： - ~35 个 Streamlit 项目（hydro-* 9 个 + audiobook/logs/auggie-dashboard/dockit/cmds/stack/playbooks/assets/…） - 3 个 Next.js（website / ops-console / mindmap） - 其余工具与配置 repo Streamlit 视觉底座（`st.sidebar` / `st.dataframe` / 默认色系）无法对齐主站 `tianlizeng.cloud` 已定型的 **Apple 风亮色 + Tailwind HSL 变量 + shadcn/ui + 4 方向 accent 色**。造成： 1. **视觉割裂**：mega navbar 通过 `~/Dev/configs/menus/navbar.yaml` SSOT 已统一，但进任一 hydro-* 子域立刻跌回 Streamlit 默认风。 2. **复制成本**：9 个 hydro-* 各自维护 `deploy.sh` + `app.py` + `plugin.yaml`，任一交互细节改一次要改 9 遍。 3. **前后端混写**：计算逻辑（AHP/TOPSIS/GeoJSON ETL/QGIS/edge-tts/Plotly）与展示层耦合在同一个 `.py` 文件里，既不利视觉统一，也挡住 API 复用。 ### 目标 1. **视觉零差异**：35 个 Streamlit 全迁 Next.js，共享 `packages/tokens` + `packages/ui`，主站视觉规则跑满所有子域。 2. **后端零重写**：Python 计算逻辑保留在原 repo 原封不动，每个 repo 套 ~30 行 FastAPI wrapper 暴露 JSON API。 3. **批量可复制**：hydro-toolkit pilot 跑通 → 抽 `stack_migrate_hydro.py` → 剩下 9 个 hydro-* 走同一 pattern。 4. **清单透明**：新建 `/stack-classify` command，一键产出 `~/Dev` 50+ 项目的技术栈 × 活跃度 × 部署 × 迁移目标清单，只读不动。 ### 已拍板决策（引用，不再议） - 视觉基准：website `tailwind.config.ts` + `app/globals.css`（HSL 变量 + 4 方向 accent + `forcedTheme="light"`，**不启用暗色模式**） - 全迁：35 个 Streamlit 全改 Next.js 前端；Python 保留原 repo - 组织：新建 `~/Dev/web-stack/` Turborepo monorepo 承载所有 Next.js apps + 共享 packages - 通信：每个 Python repo 加 FastAPI wrapper；内网 127.0.0.1:port，外网 Nginx 8443 反代，不动 CF Tunnel - Pilot：audiobook + hydro-toolkit 并行 - /tidy 不改，新建 `/stack-classify`（命名沿用 `stack.tianlizeng.cloud` 项目前缀） --- ## 1.
