# Cockpit — 个人效率驾驶舱 ## Context 你有 45+ 个项目在 ~/Dev/ 下，每天在标书、报告、工具链、资产研究之间频繁切换。最大的痛点是： 1. **"今天该做什么"没有统一入口** — 每次都手动问 cclog、查 tasks.json 2. **跨项目知识检索困难** — CC 会话里的结论散落在 711+ 条记录中 3. **外出时无法快速做水利计算** — 所有工具都是桌面端 4. **VPS 服务状态要开浏览器看 dashboard** 做一个 Tauri v2 桌面 app **Cockpit**，统一解决这些问题。 ## 架构决策 **独立项目 `~/Dev/cockpit/`**，不放进 hydro-apps monorepo。 理由：hydro-apps 是纯水利计算工具集，Cockpit 的关注点（SQLite 读取、HTTP 健康检查、任务追踪）完全不同。但会通过 path dependency 复用 hydro-apps 的计算 crate 和 React 共享组件。 ## 技术栈 | 层 | 技术 | |----|------| | 前端 | React 19 + TypeScript 5.7 + Vite 6 + Tailwind | | 图表 | Recharts 2.15 | | 桌面 | Tauri 2 | | 后端 | Rust 2021 | | 数据 | rusqlite (cclog SQLite), serde_json (tasks.json), reqwest (HTTP) | | 共享 | @hydro/shared 组件 (AppShell, DataTable 等) | ## 数据源 | 数据 | 位置 | 访问方式 | |------|------|----------| | 任务列表 | `~/Dev/configs/tasks.json` | Rust fs::read_to_string | | CC 会话 | `~/.cache/cclog/sessions.db` | rusqlite 直读 | | VPS 状态 | `*.tianlizeng.cloud` 各子域 | reqwest HTTP HEAD | | 水利计算 | hydro-apps crates | Cargo path dependency | ## 功能优先级 ### Week 1: Dashboard + Knowledge (核心价值) **Day 1: 项目脚手架** - 创建 `~/Dev/cockpit/` 目录结构 - 初始化 Tauri 2 + React 19 + Vite 6 + TypeScript - 链接 @hydro/shared，导入 global.css - 侧栏导航 shell（Dashboard / Knowledge / Calculators） - 验证 `pnpm tauri dev` 能跑 **Day 2-3: Dashboard 页面** - Rust `commands/tasks.rs`: 读取 tasks.json，返回结构化数据 - 字段：title, repo, status, next_step, resume_cmd, analyzed_at - 过滤 status == "in-progress" 的任务 - Rust `commands/cclog.rs`: 读取 `~/.cache/cclog/sessions.db` - `get_daily_digest(date)`: 昨日会话统计 + 摘要 - `get_session_stats()`: 总会话数、本周时长、活跃项目 - Rust `commands/vps.rs`: 并发 HTTP HEAD 检查各服务 URL - 超时 5s，返回 service_name + status (up/down/cf_access) - React Dashboard.tsx: - 顶部：日期 + 问候 + 本周统计卡片 - 中间：in-progress 任务列表（点击复制 resume_cmd） - 右侧：VPS 服务状态灯 - 底部：昨日 CC 会话摘要 **Day 4-5: Knowledge 页面** - Rust `commands/cclog.rs` 扩展: - `search_sessions(query, project?, date_from?, date_to?, limit)`: 全文搜索 title/summary/outcomes - `get_session_detail(session_id)`: 完整会话信息 - React Knowledge.tsx: - 搜索栏 + 项目/日期筛选 - 结果卡片：日期、项目、标题、摘要片段、时长 - 展开详情：outcomes, learnings, files_modified, tools_used ### Week 2: Hydro Calculators + Polish **Day 6-8: 水利计算器集成** - Cargo.toml 添加 hydro-apps path dependencies: - `hydro-capacity-app = { path = "../hydro-apps/apps/capacity/src-tauri" }` - `hydro-irrigation-app = { path = "../hydro-apps/apps/irrigation/src-tauri" }` - lib.rs 注册 capacity + irrigation 的 Tauri commands（参照 toolkit/lib.rs 模式） - React Calculators.tsx: - 工具卡片网格 - 复用 hydro-apps 各 app 的 React 组件（通过 Vite alias 导入） **Day 9-10: Polish + 打包** - 视觉打磨：loading states, error boundaries, 响应式布局 - Settings 页面：显示数据源路径，VPS URL 配置 - `pnpm tauri build` 生成 .dmg - 写 CLAUDE.md，git init，push to GitHub ### 后续迭代（不在 MVP 范围） - **DOCX 审阅**: 需要 DOCX→HTML 渲染器，工作量大，单独里程碑 - **iOS 移动端**: Tauri v2 支持但需要数据同步层（VPS REST API），另做规划 - **VPS 服务控制**: 需要 VPS 端 REST API，MVP 只做状态查看 ## 项目结构 ``` ~/Dev/cockpit/ ├── src/ │ ├── App.tsx # 路由 + 侧栏导航 │ ├── main.tsx │ ├── pages/ │ │ ├── Dashboard.tsx # 晨间简报 + 任务总览 │ │ ├── Knowledge.tsx # cclog 会话搜索 │ │ └── Calculators.tsx # 水利快速计算 │ ├── components/ │ │ ├── TaskList.tsx │ │ ├── VpsStatus.tsx │ │ ├── SessionSearch.tsx │ │ └── StatCard.tsx │ └── lib/ │ └── api.ts # Tauri invoke wrappers ├── src-tauri/ │ ├── Cargo.toml │ ├── tauri.conf.json │ ├── capabilities/default.json │ └── src/ │ ├── main.rs │ ├── lib.rs # Tauri builder │ ├── commands/ │ │ ├── tasks.rs # tasks.json 读取 │ │ ├── cclog.rs # cclog SQLite 查询 │ │ ├── vps.rs # HTTP 健康检查 │ │ └── hydro.rs # hydro calculator 命令导出 │ └── db.rs # SQLite 连接管理 ├── package.json ├── vite.config.ts ├── tailwind.config.js ├── tsconfig.json ├── index.html └── CLAUDE.md ``` ## 关键复用文件 | 文件 | 复用什么 | |------|----------| | `~/Dev/hydro-apps/apps/toolkit/src-tauri/src/lib.rs` | 多 app 命令聚合模式 | | `~/Dev/hydro-apps/packages/shared/` | AppShell, DataTable, ToolCard, global.css | | `~/Dev/cclog/src/cclog/indexer.py:13-50` | cclog SQLite schema 定义 | | `~/Dev/configs/tasks.json` | 任务数据结构 | | `~/Dev/hydro-apps/apps/capacity/src-tauri/` | 纳污能力计算 Rust crate | | `~/Dev/hydro-apps/apps/irrigation/src-tauri/` | 灌溉需水计算 Rust crate | ## 风险与应对 | 风险 | 应对 | |------|------| | hydro-apps path dependency 跨 repo 耦合 | 可接受，个人项目，两个 repo 都在 ~/Dev/ | | @hydro/shared link 依赖可能有传递依赖问题 | 备选：直接复制 6 个组件文件（~300行） | | CF Access 保护的 URL 返回挑战页不是 200 | 检测 CF challenge HTML 也算 "up"，或加 /health bypass 端点 | | cclog DB 可能过期 | 启动时检查 metadata.last_indexed_at，过期提示刷新 | | rusqlite bundled 编译慢 | 首次约 30s，增量编译无影响 | ## 验证方式 1. `pnpm tauri dev` 启动，Dashboard 展示 in-progress 任务 2. 点击任务，resume_cmd 复制到剪贴板 3. VPS 状态灯正确显示各服务 up/down 4. Knowledge 搜索 "hydro" 返回相关会话 5. Calculators 能加载纳污能力/灌溉需水的示例数据并计算 6. `pnpm tauri build` 成功生成 .dmg
