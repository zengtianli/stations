Plan: /groom 命令 + repo 标准体系

PLAN · (plan) · 2026-04-02 22:30 · ~/.claude/plans/vivid-growing-cherny.md
# Plan: /groom 命令 + repo 标准体系 ## Context 38 个自有 GitHub repo 分散在 ~/Dev/、~/Work/、~/.hammerspoon/。现有命令（/repo-map、/promote、/tidy、/ship）各管一块但缺少编排层。repo 质量参差：A 档 5 个全齐，B 档 16 个缺 LICENSE/topics，C 档 7 个缺 README/CLAUDE.md。 目标：一个 `/groom` 命令把 pull → audit → fix → review → push 串起来，同时建立 repo 完整性标准。 ## 要做的事 ### 1. 更新 repo-map.json ``` ~/Dev/configs/repo-map.json ``` - 新增 `hammerspoon`：`{"local": "~/.hammerspoon", "github": "zengtianli/hammerspoon", "vps": null, "category": "personal", "auto_push": false}` - 新增 `"ignored": true` 字段给不需要 groom 的 repo（extensions fork 不在 map 里，scripts/scripts-archive 不在 map 里，所以不用加） - 不加 extensions/scripts/scripts-archive/zengtianli（profile repo） ### 2. 新建 repo-standards.json ``` ~/Dev/configs/repo-standards.json ``` 定义每个 category 的完整性标准： | 维度 | common (所有 repo) | hydro/tools 额外 | infra/knowledge/personal/work | |------|-------------------|------------------|------------------------------| | LICENSE | MIT ✓ | — | — | | .gitignore | baseline entries ✓ | — | — | | README.md | 必须 | 双语 (+ README_CN.md) | 单语即可 | | CLAUDE.md | 必须 | — | — | | GitHub description | 非空 | — | — | | GitHub topics | ≥3 | base: hydrology/water-resources/python | 按类定 | | Homepage URL | 有部署则必须 | pattern: {name}.tianlizeng.cloud | — | | Screenshot | — | 必须 (docs/screenshots/demo.png) | 不需要 | README 模板分 4 种： - **app** (hydro + tools)：language selector → badges → demo/screenshot → feature table → install → quick start → license - **infra**：title → badges → what-it-does → setup → usage → license - **content** (knowledge)：title → badges → contents overview → license - **minimal** (personal + work)：title → overview → license Gold standard：`~/Dev/dockit/README.md` ### 3. 新建 groom.md 命令 ``` ~/Dev/cc-harness/commands/groom.md ``` #### 参数 - 无参数 → 所有非 ignored repo - `<name> [name2 ...]` → 指定 repo - `--check` → 只审计不修复 - `--category <cat>` → 按分类过滤 - `--skip-pull` → 跳过 pull #### Phase 1: Pull 遍历目标 repo，`git pull --ff-only`。失败的记 ⚠ 继续。输出汇总表。 #### Phase 2: Audit 读 repo-standards.json，逐个检查： - 文件：LICENSE / .gitignore / README.md / README_CN.md / CLAUDE.md - README 质量：按模板检查 sections - GitHub metadata：description / topics / homepage - 如果 `--check`，输出审计报告后停止 输出： ``` ✅ dockit — all clear 🔧 hydro-rainfall — missing LICENSE, 2 README issues 🔧 learn — missing CLAUDE.md, no LICENSE ``` #### Phase 3: Fix（按类型分组） **自动修复（批量确认）：** 1. LICENSE — MIT 模板，copyright tianli 2. .gitignore — 补 baseline entries 3. GitHub metadata — description/topics/homepage via `gh repo edit` **需要逐个审查的：** 4. CLAUDE.md — 读代码后生成（Quick Reference + 结构 + 命令） 5. README.md — 按模板生成/修复 6. README_CN.md — 从 EN 翻译 7. /tidy 清理 — 仅 work 类 repo 且有 deliverables/docs 目录时触发 #### Phase 4: Review Checkpoint **按修复类型分组展示**（不是按 repo）： ``` LICENSE 添加 (12 repos): hydro-rainfall, learn, essays, ... → 全部 MIT 模板，批量确认？[Y/n] CLAUDE.md 新建 (3 repos): → learn: [展示内容] → docs: [展示内容] → 逐个确认 README 更新 (5 repos): → hydro-rainfall: [展示 diff] → 逐个确认 GitHub metadata (8 repos): [表格: repo | 变更 | old → new] → 批量确认？[Y/n] ``` #### Phase 5: Ship 对有变更的 repo：git add 具体文件 → commit（conventional commits） → push。 最终汇总表。 ### 4. 处理 archived repos GitHub 上 scripts 已 archived，scripts-archive 是残留。这两个不在 repo-map 里，不需要处理。用户说删——在 GitHub 上 delete 这两个 repo。 ## 关键文件 | 文件 | 操作 | |------|------| | `~/Dev/configs/repo-map.json` | 更新：加 hammerspoon | | `~/Dev/configs/repo-standards.json` | **新建** | | `~/Dev/cc-harness/commands/groom.md` | **新建** | 引用但不改的： - `~/Dev/cc-harness/commands/promote.md` — groom Phase 3 metadata 逻辑引用 - `~/Dev/cc-harness/commands/tidy.md` — groom Phase 3 cleanup 逻辑引用 - `~/Dev/cc-harness/commands/ship.md` — groom Phase 5 逻辑引用 - `~/Dev/devtools/repo_manager.py` — audit 函数可复用 ## 验证 ```bash # 1. 检查新文件 cat ~/Dev/configs/repo-standards.json | python3 -m json.tool # valid JSON cat ~/Dev/cc-harness/commands/groom.md # 命令内容 # 2. repo-map 更新 python3 -c "import json; d=json.load(open('$HOME/Dev/configs/repo-map.json')); print('hammerspoon' in d['repos'])" # True # 3. 试跑 dry run /groom --check # 只看审计结果，不改任何东西 # 4. 小范围测试 /groom dockit cc-harness # 这两个已经是 A 档，应该 all clear # 5. 全量 /groom # 36 个 repo 全跑 ``` ## 不做的事 - 不改 promote.md / tidy.md / ship.md 的内容（groom 引用它们的逻辑） - 不改 repo_manager.py（后续重构时再统一） - 不处理 extensions fork（不是自有 repo）
