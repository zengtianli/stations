# ZDWP Git 清理与重建计划 ## 现状分析 当前仓库: 1763 文件, 42.7 MB 工作区, ~32 MB .git/pack 远程: https://github.com/zengtianli/zdwp.git ### 需要移除的垃圾数据 | 目录 | 文件数 | 大小 | 内容 | 决定 | |------|--------|------|------|------| | `projects/eco-flow/code/水文频率工具/` | 51 | 30.8 MB | 水库频率数据 .txt + .ini + .html | **全部移除** — 纯数值数据，auggie 无法索引 | | `resources/` | 292 | 6.4 MB | 188 个 _meta.json (水库元数据)、84 个 .py (数据处理脚本)、配置文件 | **全部移除** — GIS 元数据 JSON + 数据处理脚本不属于知识库 | | `projects/water-source-assessment/` | 715 | 0.1 MB | 主要是缙云县潜明水库的参考 .md 文件(从 docx 转换) | **保留** — 都是有效的 .md 参考资料，体积很小 | | `projects/shoreline-planning/` | 41 | < 1 MB | 岸线规划收赛资料 .md 文件 | **保留** — 有效参考资料 | | 老中文目录名 | 0 | - | 已经没有了 | 无需处理 | ### 数据说明 - `resources/` 292 文件分析: - 188 个 `_meta.json` — 婺城等地水库控运计划的元数据，结构化 GIS 数据 - 84 个 `.py` — 数据处理脚本（build_eco_survey_table.py 等）应该在 ~/Dev - 少量 yaml/md/gitignore 配置 - 这些都是数据处理产物，不是 auggie 需要索引的知识 - `projects/water-source-assessment/` 715 文件但只有 0.1 MB — 这些是合理的 .md 参考资料文档，保留 ## 执行步骤 ### Step 1: 更新 .gitignore 在 .gitignore 中添加排除规则，防止重建后重新跟踪: ``` # ── 排除数据目录 ───────────────────────────────── resources/data/ projects/eco-flow/code/水文频率工具/ ``` 注意: resources/ 下有些有用的 _index.yaml 和说明文档，但整体来看都是数据处理产物。最简单的做法是排除整个 `resources/data/` 目录（占 292 文件中的绝大部分），保留 resources/ 顶层的 _files.md 和 gis/_index.yaml、references/ 下的说明书。 实际上再看一遍，resources/ 下面： - `resources/_files.md` — 自动生成的清单，可保留 - `resources/gis/_index.yaml` — 空索引 - `resources/references/_index.yaml` — 空索引 - `resources/data/` — 绝大部分垃圾 - `resources/references/内部资料/` — 少量说明书 .md，有价值 所以排除 `resources/data/` 即可。 ### Step 2: 执行 git 重建 ```bash cd ~/Work/zdwp # 保存远程 REMOTE_URL=$(git remote get-url origin) # 删除旧历史 rm -rf .git # 创建新仓库（此时 .gitignore 已更新，会自动排除垃圾） git init git add . git commit -m "init: 数据目录模式 — 纯文本知识库供 auggie 索引" # 恢复远程 git remote add origin "$REMOTE_URL" git branch -M main git push --force origin main ``` ### Step 3: 验证 预期结果: - 移除: ~51 (水文频率) + ~285 (resources/data/) = ~336 文件 - 从 1763 → ~1427 文件 - .git/ < 10 MB（无历史包袱） ### 风险 - `git push --force` 会覆盖远程历史 - 其他 clone 需要重新 clone - 确认没有未推送的分支
