# CC Dashboard 美化计划 ## Context cc-dashboard 是 QQQ Covered Call 策略的 Streamlit 单页 Dashboard，当前使用 Streamlit 默认样式，信息密度高但视觉层次弱。参考用户个人网站（Next.js + Tailwind, Apple 风格）的设计语言，对 Dashboard 进行视觉美化和布局重构。 **目标**：Apple 风格配色 + Inter 字体 + 卡片化 KPI + Tab 导航 + 期权表拆分 + Roll 信号醒目展示 **约束**：保持 Streamlit 框架不变、所有计算逻辑不动、VPS 部署兼容 --- ## 改动文件 | 文件 | 改动类型 | |------|----------| | `cc-dashboard/app.py` | 重构（CSS注入 + Tab布局 + 表格拆分 + Plotly主题） | | `cc-dashboard/.streamlit/config.toml` | 新增 `[theme]` 配置 | 不改动：`lib_greeks.py`、数据文件、sidebar 控件逻辑 --- ## 实施步骤 ### Step 1: config.toml 主题配色 在 `.streamlit/config.toml` 添加： ```toml [theme] primaryColor = "#0071e3" backgroundColor = "#fbfbfd" secondaryBackgroundColor = "#f5f5f7" textColor = "#1d1d1f" font = "sans serif" ``` ### Step 2: CSS 注入函数 `inject_custom_css()` 在 `st.set_page_config()` 之后调用，注入 `<style>` 标签： - **字体**：`@import` Inter from Google Fonts，覆盖 Streamlit 全局字体 - **st.metric 卡片**：`[data-testid="stMetric"]` → 白底、`border-radius: 1rem`、柔和阴影、`1px solid #e5e5e7` - **Markdown 表格**：圆角、`th` 灰底、hover 行高亮、交替行色 - **DataFrame**：圆角 + 阴影 - **Tab 栏**：active tab 蓝色下划线 `#0071e3`，inactive 灰色 `#86868b` - **Sidebar**：背景 `#f5f5f7` - **Roll 信号徽章 CSS class**： - `.signal-hold` → 浅绿底 `#e8f5e9` + 深绿字 `#2e7d32` - `.signal-roll` → 浅黄底 `#fff8e1` + 深黄字 `#f57f17` - `.signal-assign` → 浅红底 `#fce4ec` + 深红字 `#c62828` ### Step 3: Plotly 统一主题 新增 `create_plotly_template()` → 注册为 `pio.templates.default`： - 字体 Inter，纸面色 `#fbfbfd` - 配色序列：`#0071e3`（CC蓝）、`#34c759`（QQQ绿）、`#ff9500`（杠杆橙） - 网格线 `#e5e5e7`，图例半透明白底 更新两处 Plotly 图表（TWR 曲线 + 收益对比柱状图）的 trace 颜色： - CC 实盘：`#0071e3` - QQQ 无杠杆：`#34c759` - QQQ 杠杆：`#ff9500` ### Step 4: Helper 函数 **`render_metric_card(label, value, subtitle=None, bg_color=None)`** - 返回 HTML 字符串，用 `st.markdown(html, unsafe_allow_html=True)` 渲染 - 白底卡片 + 圆角 + 阴影，支持 `bg_color` 条件着色 **`render_signal_card(exp, strike, qty, action, target, reason, tv_harv)`** - 返回 HTML 卡片：标题行（到期+Strike）、信号徽章、Roll 目标、理由 - TV 收割进度条（inline HTML div，宽度 = `tv_harv%`） - 根据 action 分配 CSS class：HOLD→绿、ROLL→黄、ASSIGN→红 ### Step 5: Tab 布局重构 **关键原则**：所有计算保持在 `st.tabs()` 之前，tab 内只放渲染代码。 ```python # 数据加载 + 全部计算（不变） data = load_portfolio() df_nlv = load_daily_nlv() # ... 所有 compute_* 调用 ... # 全局信息行（Tab 上方） st.caption(f"📅 {_data_date} 收盘 | QQQ ${qqq_px:.2f} | Updated {data['ts']}") tab1, tab2, tab3, tab4, tab5 = st.tabs(["总览", "风险", "期权", "绩效", "情景"]) ``` #### Tab 1 "总览" - 4 个 `render_metric_card()`：NLV、QQQ 市值、Margin 借款、杠杆倍数 - 解释表格（账户总览 + Theta/Margin/Sharpe 说明） - 第二行 4 个 metric card：Theta 收租、Margin 利息、日净收入、Sharpe #### Tab 2 "风险" - Row A：5 个 metric card（名义杠杆、有效杠杆、对冲比例、±1%影响、影响占NLV） - Row B：4 个 metric card（净Delta、净Gamma、日Theta、净Vega） - **直接展示**风险分解表（不再折叠在 expander 里）：杠杆与对冲表、Greeks 解释表、±1% P&L 分解表 #### Tab 3 "期权" - **表 A "持仓概况"**（8列）：到期、DTE、行权价、数量、现价、IV、Delta、Theta - **表 B "收益分析"**（7列）：到期、行权价、收取权利金、Roll年化、Roll杠杆年化、剩余TV年化、杠杆年化 - **Roll 信号卡片**（2-3列 grid）：每个期权一张 `render_signal_card()`，醒目显示建议/目标/理由 - Roll 计算明细 expander（保留折叠） - 指标说明 expander（保留折叠） - 3 个汇总 metric（剩余可收租金、已收权利金、年 Margin 利息） #### Tab 4 "绩效" - TWR 归一化曲线（Plotly，新配色） - 策略对比表（年化收益、Sharpe、Sortino、最大回撤） - 收益对比柱状图（日/周/月/年） #### Tab 5 "情景" - 情景 metric 卡片 grid - 解释表格（计算说明） **策略文档**（strategy-roll.md）放在 tabs 下方，保持现有折叠渲染。 ### Step 6: 期权表拆分实现 重构现有 `for r in opt_rows` 循环（app.py:402-493）： - 保持所有计算逻辑不变 - 循环末尾构建 3 个数据结构（而非 1 个 18 列 dict）： - `table_a_data`：持仓概况 8 列 - `table_b_data`：收益分析 7 列 - `signal_cards_data`：信号卡片数据 --- ## 实施顺序 1. config.toml 主题 → 2. CSS 注入 → 3. Plotly 主题 → 4. Helper 函数 → 5. Tab 布局 → 6. 表格拆分 每步完成后 app 仍可正常运行，逐步美化。 ## 验证 ```bash # 语法检查 source ~/.personal_env && python3 -c "import sys; sys.path.insert(0,'cc-dashboard'); import app" # 本地启动验证 streamlit run cc-dashboard/app.py # 检查所有 tab 渲染正常、数据与美化前一致 # 检查 VPS 部署（push 后 webhook 自动更新） ```
