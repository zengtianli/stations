Fix ops-console navbar — 参与站群共享 navbar，不再做独立品牌

PLAN · (plan) · 2026-04-17 22:25 · ~/.claude/plans/witty-wondering-honey.md
# Fix ops-console navbar — 参与站群共享 navbar，不再做独立品牌 ## Context 上一轮 ops-console pilot 上线后视觉错位：ops-console 在 `dashboard.tianlizeng.cloud` 自造了一条 navbar（"Ops Console" 品牌 + 概览/Auggie 索引/Hammerspoon/返回主站），但 stack / cmds / changelog 等站群成员共用的是 `~/Dev/devtools/lib/templates/site-navbar.html`（曾田力 · Services · Stack · CC Docs · Changelog · Dashboard，Dashboard 是其中一项高亮）。用户一眼看出 dashboard 跟其他站不是一家人。 根因：我把"UI 统一"理解成"各站各做一条长得像的 navbar"，而正解是"所有站群成员顶着同一条 DOM+CSS"。`返回主站` 链接就是最明显的 tell——说明我把 ops-console 当成外人而非 Dashboard 本人。 目标：ops-console 放弃独立 navbar，加入站群共享 navbar；把 `概览/Auggie/Hammerspoon` 降级为 ops-console 内部的第二层 sub-nav。 ## 当前站群 navbar 机制（已确认） **SSOT**：`~/Dev/devtools/lib/templates/site-navbar.html`（42 行自包含 HTML：inline `<style>` + `<nav>` + `<script>` 按 hostname 自动加 `.current` 高亮） **消费方式**：各消费者 repo 里 vendor 一份 `site-navbar.html`，其 build 脚本 "本地 → 回退 devtools 模板" 读取注入。`/navbar-refresh` 命令负责把 devtools 的 SSOT 同步到所有消费者副本。 **现消费者**： - stack / cmds / cc-evolution（`generate.py::_load_navbar()` 加载 → 塞进 `<body>` 头部） - audiobook（FastAPI routes/shelf.py + player.py 同样 load） - 3 个 Streamlit dashboard（app.py 里 `st.markdown(unsafe_allow_html=True)`，但因 Streamlit SPA 视觉上覆盖失败） **site-navbar.html 的 `<script>` 逻辑**（42 行最后一行）： ```js var h=location.hostname,k='home' if(h==='stack.tianlizeng.cloud')k='stack' else if(h==='cmds.tianlizeng.cloud')k='cmds' else if(h==='changelog.tianlizeng.cloud')k='changelog' else if(h==='dashboard.tianlizeng.cloud')k='dashboard' ← 会正确高亮 Dashboard ``` ## 步骤 ### 1. 在 ops-console vendor `site-navbar.html` `cp ~/Dev/devtools/lib/templates/site-navbar.html ~/Dev/ops-console/site-navbar.html` ### 2. 新建 `components/shared-navbar.tsx` — 把 HTML 注入成真 DOM Server component，build 时 `fs.readFileSync` 读本地 `site-navbar.html`，`dangerouslySetInnerHTML` 注入。 **关键坑**：`dangerouslySetInnerHTML` 会保留 `<style>` 元素（渲染生效），但 `<script>` 元素不会被执行（React/浏览器规范）。解决：把 script 里"按 hostname 加 current class" 的逻辑改成 SSR 时直接替换 HTML 字符串——在 `data-site="dashboard"` 的 `<a>` 上预加 `class="current"`。这样不依赖 script。 ```tsx // components/shared-navbar.tsx import { readFileSync } from "node:fs" import path from "node:path" const HTML = readFileSync(path.join(process.cwd(), "site-navbar.html"), "utf-8") // SSR 预高亮 Dashboard（因为本站固定服务于 dashboard.tianlizeng.cloud） .replace( /<a href="https:\/\/dashboard\.tianlizeng\.cloud" data-site="dashboard">/, '<a href="https://dashboard.tianlizeng.cloud" data-site="dashboard" class="current">' ) // 去掉 script（现在没用了） .replace(/<script>[\s\S]*?<\/script>/, "") export default function SharedNavbar() { return <div dangerouslySetInnerHTML={{ __html: HTML }} /> } ``` ### 3. 新建 `components/section-nav.tsx` — ops-console 内部二级 tab Client component（需要 `usePathname` 判断当前路由）。风格跟顶部 navbar 呼应：pill bar、白底 backdrop-blur、sticky 在顶部 navbar 下面。 ```tsx "use client" import Link from "next/link" import { usePathname } from "next/navigation" import { cn } from "@/lib/utils" const ITEMS = [ { href: "/", label: "概览" }, { href: "/auggie", label: "Auggie 索引" }, { href: "/hammerspoon", label: "Hammerspoon" }, ] export default function SectionNav() { const pathname = usePathname() return ( <div className="sticky top-11 z-40 bg-white/70 backdrop-blur-xl border-b border-black/5"> <div className="max-w-[1500px] mx-auto px-5 py-2 flex gap-1 text-[13px]"> {ITEMS.map((it) => { const active = it.href === "/" ? pathname === "/" : pathname.startsWith(it.href) return ( <Link key={it.href} href={it.href} className={cn( "px-3 py-1.5 rounded-md transition", active ? "bg-[#1f2328] text-white" : "text-[#6b7280] hover:bg-black/5 hover:text-[#1f2328]" )} > {it.label} </Link> ) })} </div> </div> ) } ``` ### 4. 改 `app/layout.tsx` - 删除 `import Navbar from "@/components/navbar"` - 改用 `SharedNavbar` + `SectionNav` - 调 main 的 top padding：shared navbar 44px + sub-nav ~36px ≈ 80px（加 `pt-4` 就够，因为 site-navbar.html 已经带 `body { padding-top: 56px }`） ```tsx <body> <SharedNavbar /> <SectionNav /> <main className="max-w-6xl mx-auto px-6 py-8">{children}</main> </body> ``` ### 5. 删 `components/navbar.tsx` 这个文件是错误的独立 navbar，完全废弃。 ### 6. 更新 `/navbar-refresh` 消费者列表 `~/Dev/devtools/scripts/tools/navbar_refresh.sh` 的 `CONSUMERS` 数组加 `~/Dev/ops-console`。下次改 navbar 模板时，ops-console 跟其他 4 站一起同步。 ### 7. `pnpm build` + `bash deploy.sh` ### 8. 视觉验证 - 打开 https://dashboard.tianlizeng.cloud 和 https://stack.tianlizeng.cloud，对比顶部 navbar：品牌、链接集、字体、颜色、间距、高亮样式（Dashboard/Stack 各自反白）应**pixel-perfect 一致** - dashboard 顶栏下面一行是 ops-console 自己的 sub-nav（3 个 pill：概览/Auggie 索引/Hammerspoon） - 浏览 /auggie /hammerspoon 路由时，顶栏 Dashboard 始终高亮；sub-nav 各自高亮 - `curl -s https://dashboard.tianlizeng.cloud | grep -c "tlz-nav"` ≥ 1 ## 关键文件 **新增** - `~/Dev/ops-console/site-navbar.html`（vendor 副本） - `~/Dev/ops-console/components/shared-navbar.tsx` - `~/Dev/ops-console/components/section-nav.tsx` **修改** - `~/Dev/ops-console/app/layout.tsx` - `~/Dev/devtools/scripts/tools/navbar_refresh.sh`（`CONSUMERS` 数组加第 5 项） **删除** - `~/Dev/ops-console/components/navbar.tsx` ## 复用 - `~/Dev/devtools/lib/templates/site-navbar.html` — 站群 navbar SSOT - `~/Dev/stack/generate.py::_load_navbar()`（第 22-28 行）— "local-first, fallback to devtools template" 加载模式的参考 - `/navbar-refresh` — vendor 副本同步机制 ## 风险 | 风险 | 对策 | |---|---| | `<style>` 在 `dangerouslySetInnerHTML` 里会有 React warning | 可接受（SSR 阶段生效，警告不影响渲染）。彻底无警告版本：提取 CSS 到 globals.css + React 组件重写 nav DOM。本轮先接受警告 | | Script 不执行 → 高亮失效 | 已通过 SSR 字符串预替换解决（data-site="dashboard" 直接加 class="current"）| | sub-nav 跟 shared navbar 背景叠加不好看 | section-nav 用 `bg-white/70 backdrop-blur-xl`，跟 site-navbar 的 `rgba(255,255,255,0.85) blur(20px)` 协调；`sticky top-11`（44px）紧贴 | | 老 ops-console 的 `components/navbar.tsx` 还有被引用 | grep 整个项目确认只有 layout.tsx 引用，删除安全 | ## 非目标（明确不做） - **抽共享 navbar 到 npm 包**：等有第 3 个 Next.js 站（当前只有 website + ops-console，website 还用另一条 portfolio-专属 nav）再动。现在抽是过早抽象。 - **把 site-navbar.html 改写成纯 React 组件**：破坏"一个 HTML 文件服务所有 Python/FastAPI/Next.js 消费者"的 SSOT 模型。保持 HTML 注入法，4 个 Python 消费者零改动。 ## 验证（端到端） 1. 本地 `pnpm dev` 打开 localhost:3007，顶栏跟 `https://stack.tianlizeng.cloud` 同款 2. `pnpm build` 绿 3. `bash deploy.sh` 全流程走通，HTTP 200 4. 浏览器交叉对比：stack 和 dashboard 两个 tab 并排看，顶栏无视觉差异；各自高亮自己那项 5. 点 dashboard 站的 Stack 链接，跳到 stack.tianlizeng.cloud，顶栏 Stack 高亮 6. `/navbar-refresh --dry-run` 能识别 ops-console 为新消费者
